## Current state

`HexBerlekampZassenhausMathlib/IntReductionMod.lean` defines the two
fast-path **small-mod singleton arm** per-branch umbrellas of the
HO-1 substrate for the capstone `factor_irreducible_of_nonUnit`
(#4170):

* `factor_small_mod_singleton_branch_entry_irreducible_of_choosePrimeData`
  (line 2696, #4564) — `_of_choosePrimeData` form;
* `factor_small_mod_singleton_branch_entry_irreducible`
  (line 2771, #4605) — `hchoose`-free form, internally derives
  `hchoose` from `hsmall_chosen.1.isSome` and forwards to the
  `_of_choosePrimeData` umbrella above.

Both currently take
`hcomplete : Hex.reassemblyExpansionComplete (Hex.normalizeForFactor f)
    #[(Hex.normalizeForFactor f).squareFreeCore]`
as an explicit hypothesis. The non-monic singleton discharger
`reassemblyExpansionComplete_singleton_of_irreducible_of_pos_lc`
(line 2317, PR #4961 / closes #4956) now produces exactly that
conclusion from `f ≠ 0`, the squareFreeCore irreducibility witness
that both umbrellas already derive internally, and the two
universally-available facts:

* `0 < Hex.DensePoly.leadingCoeff (Hex.normalizeForFactor f).squareFreeCore`
  — supplied by `Hex.squareFreeCore_leadingCoeff_pos_of_ne_zero`
  (`HexBerlekampZassenhaus/Basic.lean:12000`);
* `0 < (Hex.normalizeForFactor f).squareFreeCore.degree?.getD 0`
  — derived from the umbrellas' existing `hdeg : ... ≠ 0` premise
  via `Nat.pos_of_ne_zero`.

So the `hcomplete` premise on these two umbrellas is redundant.

This brings the singleton arm into parity with the existing
discharger-internal pattern:

* fast-path **constant arm**
  `factor_constant_branch_entry_irreducible_of_choosePrimeData`
  (line 2856) discharges `hcomplete` internally via
  `Hex.reassemblyExpansionComplete_constant_of_ne_zero` (#4585 /
  PR #4598);

  `factor_quadratic_branch_entry_irreducible_of_quadraticRoots`

  `factor_slow_quadratic_branch_entry_irreducible_of_choosePrimeData`

After this issue lands, three of the four non-BHKS per-branch
umbrellas (constant, singleton, both quadratic) discharge
