## Deliverables

In `HexBerlekampZassenhaus/Basic.lean`, immediately following
`expandRepeatedPartFactorArray_pow_singleton` (line 11872–11897),
add the `_of_pos_lc` sibling:

```lean
theorem expandRepeatedPartFactorArray_pow_singleton_of_pos_lc
    (q : ZPoly) (k : Nat)
    (hq_pos_lc : 0 < DensePoly.leadingCoeff q)
    (hq_degree : 0 < q.degree?.getD 0)
    (hq_irr : ZPoly.Irreducible q)
    (rp : ZPoly) (hrp : rp = Factorization.factorPower q k)
    (hfuel : k + 1 ≤ rp.size + 1) :
    (expandRepeatedPartFactorArray rp #[q]).2 = 1
```

Note the **weakened conclusion**: only the residual projection
`.2 = 1`, not the full pair. The full-pair version of the monic
leaf has no non-monic counterpart at the executable layer (the
`consumeExactPower_pow_mul_of_not_dvd` chain it routes through is
genuinely monic-only — under non-monic `q`, the recursive
`consumeExactPower` step's quotient is not in general a power of
`q`, even if the residual collapses to `1`). The residual-only
form is sufficient for the mid-layer consumer, which unfolds
`reassemblyExpansionComplete` to `(expand ...).2 = 1`.

The proof routes through the existing array-level non-monic public
surface
`expandRepeatedPartFactorArray_residual_eq_one_of_factorPower_decomposition_of_pos_lc`
(`HexBerlekampZassenhaus/Basic.lean:11777`, #4778) specialised to:

* `coreFactors = #[q]`,
* `exponents = [k]`,
* `hpos_lc` discharged from `hq_pos_lc` (the single-element list),
* `hdegree` discharged from `hq_degree`,
* `hlen` is trivial (`[k].length = 1 = #[q].size`),
* `hnot_dvd_tail` for the singleton split `pre = []`, `(q, k)`,
  `suf = []`: the tail product is `1`, so `¬ q ∣ 1` follows from
  the existing private helper `irreducible_not_dvd_one hq_irr`
  (line 11809). (For any non-trivial split the precondition is
  vacuously satisfied because the zip has length 1.)
* `hdecomp` is the singleton fold:
  `rp = (([(q, k)]).map (fun qe => factorPower qe.1 qe.2)).foldl (· * ·) 1`,
  which collapses to `rp = 1 * factorPower q k = factorPower q k`
  and is discharged by `hrp` combined with `DensePoly.one_mul_zpoly`
  or `Hex.ZPoly.one_mul_zpoly`.
* `hfuel'` for `(q, k) ∈ [(q, k)]`: `k + 1 ≤ rp.size + 1`, which is
  exactly `hfuel`.

The `irreducible_not_dvd_one` helper is `private` in the same file
(line 11809); the proof of the new sibling is in the same file and
thus can reference it directly.

No new `sorry`, `axiom`, `native_decide`, `TODO`, or `FIXME`.
