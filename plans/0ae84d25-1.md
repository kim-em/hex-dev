## Deliverables

In `HexGfqField/Operations.lean`, **and nowhere else**:

1. **Add `toQuotient_div`** immediately after `toQuotient_pow`
   (around line 652). Mirror the unconditional shape used by
   `toQuotient_mul` and `toQuotient_pow`:

   ```lean
   @[simp] theorem toQuotient_div
       {f : FpPoly p} {hf : 0 < FpPoly.degree f}
       {hirr : FpPoly.Irreducible f}
       (x y : FiniteField f hf hp hirr) :
       (x / y : FiniteField f hf hp hirr).toQuotient =
         x.toQuotient * (inv y).toQuotient :=
     rfl
   ```

   The proof is `rfl` because `GFqField.div_eq_mul_inv` already
   establishes the unfolding to `mul x (inv y)`, and
   `toQuotient_mul` then closes the goal definitionally. If the
   exact `rfl` fails (e.g. because of the simp normal form of
   `toQuotient_mul`), use the one-line composition
   `simpa using toQuotient_mul x (inv y)` or fall back to
   `by rw [GFqField.div_eq_mul_inv]; exact toQuotient_mul ..`.
   The worker may pick the form that elaborates without
   regressing the existing simp set.

2. **Add `toQuotient_zpow`** immediately after `toQuotient_div`.
   The `zpow` body is a pattern match on `Int.ofNat` vs
   `Int.negSucc`; the natural Lean shape is

   ```lean
   @[simp] theorem toQuotient_zpow_ofNat
       {f : FpPoly p} {hf : 0 < FpPoly.degree f}
       {hirr : FpPoly.Irreducible f}
       (x : FiniteField f hf hp hirr) (n : Nat) :
       (x ^ (Int.ofNat n) : FiniteField f hf hp hirr).toQuotient =
         x.toQuotient ^ n :=
     rfl

   @[simp] theorem toQuotient_zpow_negSucc
       {f : FpPoly p} {hf : 0 < FpPoly.degree f}
       {hirr : FpPoly.Irreducible f}
       (x : FiniteField f hf hp hirr) (n : Nat) :
       (x ^ (Int.negSucc n) : FiniteField f hf hp hirr).toQuotient =
         (inv (pow x (n + 1))).toQuotient :=
     rfl
   ```

   The pair matches the constructor split of `zpow` and stays
   `rfl`-provable. If the worker can collapse them into a single
   `toQuotient_zpow (x : ...) (i : Int) : (x ^ i).toQuotient = ...`
   form that still elaborates with a clean RHS (no metavariables),
   prefer the unified form. If not, keep the pair.

3. No changes to `def inv`, `def div`, `def zpow`, or any other
   existing definition. No new imports.
