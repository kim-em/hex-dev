`private theorem zpoly_dvd_trans` at `Basic.lean:2069` that

The audit's classification under Invariant 1 surfaces an
analogous `Hex.FpPoly p`-level divisor-transitivity cluster
that `zpoly_dvd_trans` cannot cover because `Hex.FpPoly p`

already documents the obstruction:

```
-- `dvd` instance for `FpPoly p` is `instDvdOfAddOfMul`, not the
-- default `semigroupDvd`, and `dvd_trans` doesn't see through that.
```

#5066 / #5070 cluster). The audit at
[#5069 comment](https://github.com/kim-em/hex/issues/5069#issuecomment-4480803218)

  bespoke `Mul`-derived `Dvd` instance, witness via
  `Hex.DensePoly.mul_assoc_poly (S := Int)`. 38 use sites
  routed.

  bespoke `instDvdOfAddOfMul` `Dvd` instance, witness via

  confirmed; 2 calc-form candidates need classification.

#5048 → #5069 for the ZPoly cluster) can verify cluster closure
with the same three-invariant pattern (call-site coverage,
