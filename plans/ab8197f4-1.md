## Deliverables

1. In `HexHensel/Multifactor.lean`, add `@[simp]` to
   `Hex.ZPoly.one_mul_zpoly`. This is the basic left-identity normaliser
   for `ZPoly` multiplication; making it part of the default simp set
   removes the need for downstream `rw [one_mul_zpoly]` calls (those
   continue to work after the change).

2. Review the head-prepend specialisations
   `Hex.ZPoly.polyProduct_singleton_append` and
   `Hex.ZPoly.polyProduct_cons_toArray` for `@[simp]` candidacy.
   Both rewrite head-prepended product forms into a left-multiplication
   with the tail product. Add `@[simp]` if and only if doing so does not
   create a loop with the existing `polyProduct_append` / `polyProduct_singleton`
   simp lemmas. Document the decision (annotate or explicitly leave
   un-annotated with a one-line rationale in the docstring) for each of the
   two theorems.

3. Review `Hex.ZPoly.list_foldl_mul_eq_mul_foldl_one`. It is a public
   helper used internally and by `HexBerlekampZassenhaus/Basic.lean` and
   `HexBerlekampZassenhausMathlib/IntReductionMod.lean`. Do **not** add
   `@[simp]` (it is not a normalisation rewrite — both sides contain the
   same `foldl`). Optionally tighten the docstring to clarify that this is
   an associativity helper for callers building `polyProduct` invariants.
