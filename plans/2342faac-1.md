## Current state

`HexBerlekampZassenhausMathlib/IntReductionMod.lean` defines the two
quadratic-arm per-branch umbrellas of the HO-1 substrate for the
capstone `factor_irreducible_of_nonUnit` (#4170):

* `factor_quadratic_branch_entry_irreducible_of_quadraticRoots`
  (line 2929, fast-path quadratic arm, #4571);
* `factor_slow_quadratic_branch_entry_irreducible_of_choosePrimeData`
  (line 3025, slow-path quadratic arm, #4575).

Both currently take `hcomplete : Hex.reassemblyExpansionComplete
(Hex.normalizeForFactor f) coreFactors` as an explicit hypothesis.
The non-monic discharger
`reassemblyExpansionComplete_quadraticIntegerRootFactors_of_ne_zero`
(line 2472, PR #4747) already produces exactly that conclusion from
`f ≠ 0` and the quadratic-arm witness
`hquad : Hex.quadraticIntegerRootFactors? (Hex.normalizeForFactor f).squareFreeCore = some coreFactors`,
which both umbrellas already carry as `hf_ne` and `hquad`. So the
`hcomplete` premise on these two umbrellas is redundant.

The sibling **fast-path constant arm umbrella**
`factor_constant_branch_entry_irreducible_of_choosePrimeData` (the
discussion starting at line 2810) **already** discharges its
`hcomplete` premise internally via
`Hex.reassemblyExpansionComplete_constant_of_ne_zero` (#4585 /
PR #4598). The docstring there explicitly states the umbrella "no
longer requires an explicit `hcomplete` premise" after that rewiring.
This issue applies the same pattern to the two quadratic-arm
umbrellas, completing the symmetry.

Neither umbrella has any in-tree consumers today:


## Context

* **Pattern precedent**: the constant-arm umbrella
  `factor_constant_branch_entry_irreducible_of_choosePrimeData`
  (discussion at line 2810; theorem body follows) already discharges
  `hcomplete` internally via
  `Hex.reassemblyExpansionComplete_constant_of_ne_zero` (#4585 /
  PR #4598). This issue applies the same idiom to the two
  quadratic-arm umbrellas now that the corresponding non-monic
  quadratic-arm discharger #4747 is in place.
* **Discharger details**: the discharger at line 2472 takes only
  `f : Hex.ZPoly`, `hf : f ≠ 0`, and `hquad`, and produces
  `Hex.reassemblyExpansionComplete (Hex.normalizeForFactor f)
  coreFactors`. The proof is non-monic (it routes through
  `expandRepeatedPartFactorArray_residual_eq_one_of_factorPower_decomposition_of_pos_lc`,
  #4778), so it handles the `2X + 3`-style primitive non-monic
  residuals that `quadraticIntegerRootFactors?` may emit.
* **No interaction with #4880**: the quadratic-arm umbrellas have
  always been monicness-free (they take primitivity via the #4545
  bridge internally, not monicness), so this is unrelated to the
  exhaustive-arm umbrella's Gap 1.
