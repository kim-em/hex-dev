core of the scaled Yun product identity in
`HexPolyFp/SquareFree.lean`:

- `yunFactorsContributionWithLevel_pow_step_algebra` — the
  single-iteration algebraic step.
- `yunFactorsLevelCompletes` — a recursive predicate witnessing
  that the scaled Yun loop terminates by `isOne c = true` within
  the available fuel.
- `yunFactorsContributionWithLevel_pow_invariant_of_completes` —
  the conditional product invariant: given
  `yunFactorsLevelCompletes c w base level fuel`, the loop's
  contribution-times-residual-power identity recovers
  `pow c (base * level) * pow w base`.

What is still missing is the unconditional version: for the actual
inputs that the square-free path uses, the
`yunFactorsLevelCompletes` predicate holds. The standard argument
is the loop's size-decreases invariant.

## Deliverables

1. In `HexPolyFp/SquareFree.lean`, prove that for derivative-active
   `f` with `f.size < fuel + 1`, the scaled Yun loop's initial state
   satisfies `yunFactorsLevelCompletes`. A workable shape is:
   ```lean
   private theorem yunFactorsLevelCompletes_of_derivative_active
       (hp : Hex.Nat.Prime p) (f : FpPoly p) (multiplicity fuel : Nat)
       (hmultiplicity : 0 < multiplicity) (hfuel : f.size < fuel + 1)
       (hzero : f.isZero = false)
       (hdf : (DensePoly.derivative f).isZero = false) :
       let g := DensePoly.gcd f (DensePoly.derivative f)
       let c := f / g
       yunFactorsLevelCompletes c g multiplicity 1 fuel
   ```
   Exact name and shape are at the implementer's discretion.

2. Compose the termination lemma with the conditional invariant to
   produce the unconditional public theorem expected by parent
   #3153 (i.e. an `…_pow_invariant` form without a
   `yunFactorsLevelCompletes` hypothesis).

3. The termination argument: each Yun iteration is one of
   - `isOne c = true`: loop returns immediately (base case).
   - `c ∤ w` (`y = gcd c w` properly divides `c`): the new `c' =
     c / y` has `c'.size < c.size`, so `c.size` strictly decreases.
   - `c | w` (`y = c`): the new state has `c' = isOne` (`c / c =
     1`), so the next iteration terminates.

   Combined with the invariant `c.size + w.size ≤ f.size + 1`
   (preserved as the loop only divides one component by a factor
   of the other), the size of `c` is bounded by `f.size`, so the
   loop completes within `fuel` iterations.

4. No `axiom`, `native_decide`, `TODO`, `FIXME`, or theorem-level
   `sorry`. Keep gcd, `pthRoot`, `weightedProduct`, and the scaled
   executable square-free definitions unchanged except for
   proof-oriented helper lemmas.
