## Current state

`HexGramSchmidt/Update.lean` exposes the integer adjacent-swap update API
(`adjacentSwap`, `adjacentSwapDenom`, `adjacentSwapPivotCoeff`,
`adjacentSwapGramDetNumerator`, etc.) for use by the HexLLL `swapStep`
witness fields. Issue #3535 landed the rational-layer basis and
coefficient identities (`basis_adjacentSwap_*`, `coeffs_adjacentSwap_*`)
but explicitly left the integer Gram-determinant and integer
scaled-coefficient adjacent-swap theorems unproved.

`HexGramSchmidt/Update.lean` currently contains ten `sorry`s in this
adjacent-swap cluster (lines 400, 410, 416, 423, 430, 437, 452, 458,
470, 476). The three named in the deliverables below are the
"transport-only" subset: they relate entries below the pivot column,
where the Gram determinant is unchanged and the coefficient identity
from the rational layer transfers directly to the integer scaled form.
The remaining seven sorries require the integer divisibility argument
through `adjacentSwapDenom` and are out of scope here.

The proof template is already in this file: `scaledCoeffs_sizeReduce_lower`
(around line 164) and `scaledCoeffs_sizeReduce_other_row` (around line
213) lift rational coefficient identities to the integer scaled form via
`scaledCoeffs_eq_fin` and `intCast_rat_injective`, using a `gramDet`
invariance lemma (`gramDet_sizeReduce`) plus the rational coefficient
identity (`coeffs_sizeReduce_lower`). The adjacent-swap analogues
proceed the same way.

This unblocks downstream HexLLL `swapStep` `ν_eq` field work for entries
strictly below the swap pivot (the cases `j.val + 1 < k.val` in the
`swapStep` field obligation in `HexLLL/Basic.lean` lines 520-525).

## Deliverables

In `HexGramSchmidt/Update.lean`, replace the `sorry` in these three
theorems with real proofs:

1. `gramDet_adjacentSwap_of_ne` (around line 397):

   ```lean
   theorem gramDet_adjacentSwap_of_ne (b : Matrix Int n m) (k : Fin n) (hk : 0 < k.val)
       (t : Nat) (ht : t ≤ n) (htk : t ≠ k.val) :
       gramDet (adjacentSwap b k hk) t ht = gramDet b t ht
   ```

   Proof sketch: case-split on `t ≤ km1.val` (rows `0..t-1` untouched,
   so the leading prefix matrix is unchanged) and `t > k.val` (the
   swap permutes rows `km1` and `k`, both inside `0..t-1`, leaving
   the Gram matrix invariant up to a row/column permutation whose
   determinant is `±1` and which squares away in the Gram product).
   Use existing `GramSchmidt.Int.leadingGramMatrixInt`/`gramDet`
   bridges and the rational `basis_adjacentSwap_*` row identities to
   show the Gram matrix at depth `t` is unchanged.

2. `scaledCoeffs_adjacentSwap_lower_prev` (around line 418):

   ```lean
   theorem scaledCoeffs_adjacentSwap_lower_prev (b : Matrix Int n m)
       (k : Fin n) (hk : 0 < k.val) (j : Fin n) (hj : j.val + 1 < k.val) :
       let km1 := GramSchmidt.prevRow k hk
       GramSchmidt.entry (scaledCoeffs (adjacentSwap b k hk)) km1 j =
         GramSchmidt.entry (scaledCoeffs b) k j
   ```

   Proof sketch: follow `scaledCoeffs_sizeReduce_lower`. Apply
   `intCast_rat_injective`, then `scaledCoeffs_eq_fin` to both sides
   to reduce to a rational identity:

   - `gramDet (adjacentSwap b k hk) (j.val + 1) = gramDet b (j.val + 1)`
     from deliverable 1 (since `j.val + 1 < k.val ≠ k.val`).
   - `coeffs (adjacentSwap b k hk)` entry at `(km1, j)` equals
     `coeffs b` entry at `(k, j)` from `coeffs_adjacentSwap_lower_prev`.

   Combine these to identify `scaledCoeffs (adjSwap) km1 j` with
   `scaledCoeffs b k j`.

3. `scaledCoeffs_adjacentSwap_lower_curr` (around line 425):

   ```lean
   theorem scaledCoeffs_adjacentSwap_lower_curr (b : Matrix Int n m)
       (k : Fin n) (hk : 0 < k.val) (j : Fin n) (hj : j.val + 1 < k.val) :
       let km1 := GramSchmidt.prevRow k hk
       GramSchmidt.entry (scaledCoeffs (adjacentSwap b k hk)) k j =
         GramSchmidt.entry (scaledCoeffs b) km1 j
   ```

   Proof sketch: same template as deliverable 2, swapping the role
   of `km1` and `k`. Use `coeffs_adjacentSwap_lower_curr` for the
   coefficient identity.
