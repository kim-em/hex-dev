#4908's signature takes `(B' : Nat) (hvalid : ∀ i, (f.coeff i).natAbs
≤ B') (hprecision)` — a single-factor bound over the extracted factor
`f`. #4924's signature takes
`(B' : Nat) (hvalid : ∀ g, toPolynomial g ∈ normalizedFactors
(toPolynomial (recombinationCandidate d T)) → ∀ i, (g.coeff i).natAbs
≤ B') (hprecision)` — a per-normalised-factor bound over the
candidate. So the `_of_bound` cover-at-min sibling needs the
per-normalised-factor bound, plus a derivation of the single-factor
bound for the extracted `f`. The natural derivation: `f` is
irreducible and divides `recombinationCandidate d T`, so `f` (after
normalisation) lies in `normalizedFactors (toPolynomial
(recombinationCandidate d T))` and `hvalid` instantiates to the
required single-factor bound.

The natural `_of_bound` shape:

```lean
(B' : Nat)
(hvalid : ∀ g : Hex.ZPoly,
  HexPolyZMathlib.toPolynomial g ∈
    UniqueFactorizationMonoid.normalizedFactors
      (HexPolyZMathlib.toPolynomial (recombinationCandidate d T)) →
  ∀ i, (g.coeff i).natAbs ≤ B')
(hprecision : 2 * B' < d.p ^ d.k)
```

This mirrors #4902 / #4924 (and avoids the scaled-only
`hcore_lc_bound` parameter since the unscaled recombinationCandidate
substrate does not consume `(leadingCoeff core).natAbs`).

The wrapper instantiates `B' := defaultFactorCoeffBound core` and
discharges `hvalid` for an arbitrary normalised factor `g` via the
divisor chain `g ∣ recombinationCandidate d T ∣ target ∣ core` plus
`defaultFactorCoeffBound_valid core hcore_ne g hg_dvd_core` — the
same discharge block used in #4902's monic wrapper rewiring.
