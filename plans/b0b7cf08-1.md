## Current state

The fast-path constant arm umbrella
`HexBerlekampZassenhausMathlib/IntReductionMod.lean:1123`
(`factor_constant_branch_entry_irreducible_of_choosePrimeData`, #4565
via PR #4572 / refined by PR #4583) takes
`hcomplete : reassemblyExpansionComplete (normalizeForFactor f) #[(normalizeForFactor f).squareFreeCore]`
as an explicit hypothesis. The HO-1 capstone
`factor_irreducible_of_nonUnit` (#4170) cannot consume the umbrella for
its constant branch until that hypothesis can be discharged from the
executable invariants alone (i.e. from `f ≠ 0` plus the constant-branch
marker `(normalizeForFactor f).squareFreeCore.degree?.getD 0 = 0`).

Two facts make this discharger tractable:

* `Hex.squareFreeCore_eq_one_of_constant_of_ne_zero`
  (`HexBerlekampZassenhaus/Basic.lean:8062`) collapses the
  square-free core to `1` in the constant branch.
* `Hex.expandRepeatedPartFactorArray_singleton_one`
  (`HexBerlekampZassenhaus/Basic.lean:7996`) computes
  `expandRepeatedPartFactorArray rp #[1] = (#[], rp)`, so
  `reassemblyExpansionComplete d #[1]` reduces to `d.repeatedPart = 1`.

What is missing is the supporting structural fact: when the square-free
core has degree zero, `(normalizeForFactor f).repeatedPart = 1`. The
private `repeatedPart` field is built from
`primitiveSquareFreeDecomposition` applied to `extractXPower
(primitivePart f) |>.core`; in the constant branch that core itself is
a constant (degree zero), so the primitive square-free decomposition's
repeated part is forced to `1`. Once that helper lemma exists,
`reassemblyExpansionComplete (normalizeForFactor f) #[1]` follows in
one rewrite via `expandRepeatedPartFactorArray_singleton_one`, and the
discharger is then a one-line composition of the two existing lemmas
plus the new helper.

This is a per-branch sibling of #4561's `hcomplete`-style substrate
pieces: the constant branch needs no `choosePrimeData` invariants at
all (the constant arm umbrella does not invoke `choosePrimeData`), so
this discharger is the simplest entry point of the four singleton-core
arms (constant, small-mod singleton, slow quadratic, fast quadratic)
that all wait on a `reassemblyExpansionComplete` discharger. The
small-mod singleton / quadratic dischargers are out of scope here and
are separate issues.

   `private theorem normalizeForFactor_repeatedPart_eq_one_of_constant`
   in `HexBerlekampZassenhaus/Basic.lean` (immediately adjacent to
   `squareFreeCore_eq_one_of_constant_of_ne_zero`, around line 8062):

   ```lean

       (f : ZPoly) (hf : f ≠ 0)
       (hdeg : (normalizeForFactor f).squareFreeCore.degree?.getD 0 = 0) :
       (normalizeForFactor f).repeatedPart = 1
   ```

   The proof should mirror the structure of
   `squareFreeCore_eq_one_of_constant_of_ne_zero` and route through the
   appropriate `primitiveSquareFreeDecomposition_repeatedPart_*_of_degree_zero`
   companion lemma in the `ZPoly` namespace. If no such companion
   exists, prove it alongside (in `HexPolyZ` or whichever namespace
   hosts the squareFreeCore degree-zero lemma already cited). Keep the
   helper `private` if its only consumer is the discharger below.

2. Add the public discharger
   `theorem reassemblyExpansionComplete_constant_of_ne_zero`
   in `HexBerlekampZassenhaus/Basic.lean` (next to the helper from
   deliverable 1):

   ```lean
   theorem reassemblyExpansionComplete_constant_of_ne_zero
       (f : ZPoly) (hf : f ≠ 0)
       (hdeg : (normalizeForFactor f).squareFreeCore.degree?.getD 0 = 0) :
       reassemblyExpansionComplete (normalizeForFactor f)
         #[(normalizeForFactor f).squareFreeCore]
   ```

   Proof: rewrite `squareFreeCore = 1` via
   `squareFreeCore_eq_one_of_constant_of_ne_zero`, unfold
   `reassemblyExpansionComplete`, compute via
   `expandRepeatedPartFactorArray_singleton_one` so the obligation

   with the helper from deliverable 1. Total proof body should be
   under ~10 lines.
