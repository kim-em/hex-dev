`HexPolyFp/Frobenius.lean` (508 lines) ships the executable
`X^p mod f` and `X^(p^k) mod f` API that `HexBerlekamp.DistinctDegree`
and `HexHensel` consumers depend on:

- `powModMonic base f hmonic n` — square-and-multiply modular
  exponentiation in `F_p[x] / (f)` (the production hot path)
- `powModMonicLinear` — structurally recursive variant for
  kernel-reducible small closed terms
- `frobeniusXMod f hmonic` — `X^p mod f`, the basic Frobenius
  generator
- `frobeniusXModLinear` — kernel-reducible variant
- `frobeniusXPowMod f hmonic k` — `X^(p^k) mod f` for arbitrary `k`
- `frobeniusXPowModLinear` — kernel-reducible variant
- `frobeniusXPowMod_zero` (`@[simp]`) — base case at `k = 0`
- `linearPow_mod_eq_of_mod_eq_mod` — invariant lifting
- `powModMonic_pos_self_mod`, `powModMonic_mod_eq_linearPow`,
  `powModMonicLinear_eq_powModMonic` — bridge between the two
  implementations
- `frobeniusXModLinear_eq_frobeniusXMod`,
  `frobeniusXPowModLinear_eq_frobeniusXPowMod` — bridge for the
  Frobenius wrappers
- `frobeniusXPowMod_succ` — successor-step (used by
  `HexBerlekamp.DistinctDegree`'s bucket invariant)
- `frobeniusXPowMod_mod_eq_monomial_mod`,
  `frobeniusXPowMod_mod_self`

## Library placement

Target file: `HexPolyFp/Frobenius.lean` (already exists).

SPEC §: `SPEC/Libraries/hex-poly-fp.md` "Contents" pins this file's
responsibilities:

> - Frobenius map: `X^p mod f` via repeated squaring
> - `X^(p^k) mod f` for arbitrary k (square-and-multiply on
>   polynomials)

Placement questions:

1. **SPEC §:** the lines quoted above pin this file directly.
2. **Mathlib?** No. HexPolyFp is a Mathlib-free library and the
   Frobenius surface is implemented over `DensePoly (ZMod64 p)`
   with the executable `modByMonic` primitive.
3. **Already in Mathlib?** No — Mathlib's `frobenius` lives in the
   abstract `CommRing` API; the executable `X^(p^k) mod f` cycle
   over `FpPoly p` is hex-specific.
4. **Missing infrastructure?** No — every declaration named here
   already exists; this is a polish review, not an addition.
