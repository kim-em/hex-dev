## Library placement

- **File:** `HexGfqRing/Operations.lean` (the nsmul / intCast / zsmul
  cluster, roughly lines 698–903 in the current revision).
- **Governing SPEC §:** `SPEC/Libraries/hex-gfq-ring.md` "Contents":

  > Ring operations: addition, multiplication, negation, subtraction,
  > exponentiation; every operation reduces via `reduceMod`.
  > `pow x n` is square-and-multiply (`O(log n)` quotient-ring
  > multiplications). `nsmul n x` and `natCast n` use binary
  > decomposition with the same complexity; the textbook
  > `n+1 ↦ pred + 1` recursion is forbidden.

- **Mathlib?** No. HexGfqRing is a Mathlib-free library and this
  cluster is the canonical-representative quotient type's native
  scalar-action and integer-cast plumbing.
- **Already in Mathlib?** No — Mathlib's quotient-ring API uses
  `Quot`/`Setoid`, and its nsmul / intCast / zsmul plumbing goes
  through `Quot.ind` and the generic `AddGroup` derivation, not a
  canonical-representative subtype with explicit recurrences.
- **Missing infrastructure?** No — every declaration named here
  already exists; this is a polish review, not an addition.
