## Context

This is a review issue. The audit pattern was introduced by
#5031, refined through #5048 → #5062 (DEVIATION) → #5066 +
#5070 → #5069 → #5076 (PASS), and is now the project's
standard three-invariant template for cluster closure.

#5084 is the first **inversion** in the cluster cleanup wave —
it moves bodies into helpers and collapses an umbrella into a
projection bundle, rather than extracting a fresh helper. The
audit shape is the same (call-site coverage / framing / cross-
file consistency), but Invariant 1's check is "no callers
regressed" rather than "all candidate sites rewired".

The post-#5054 feasibility audit (closed by audit comment on
#5050) confirmed the umbrella has zero non-corollary callers —
so the bundle exists purely to be destructured by the two
helpers. After #5084, the bundle becomes a 3-line constructor
returning the same triple but the work moves to the helpers
themselves.
