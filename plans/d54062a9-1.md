## Current state

`HexBerlekampZassenhausMathlib/Basic.lean` contains two structurally
identical inline derivations of `(Hex.ZPoly.modP factor).isZero = false`
from an `Hex.FpPoly p`-level divisibility hypothesis. The pattern
appears at:

| # | Site                 | Block lines | Enclosing theorem |
| - | -------------------- | -----------:| ------------------ |
| 1 | `Basic.lean:1318`    | 1317–1336   | `monicModPImage_dvd_monicModularImage_of_dvd_of_choosePrimeData?_some` (line 1287) |
| 2 | `Basic.lean:15442`   | 15428–15449 | (inside `monicModularImage_eq_factorsM_prod_of_*` family — confirm at PR time) |

Both blocks share the canonical shape: start from `factor ∣ core` at
the `Hex.FpPoly p` level (`hfactor_dvd_core` / `hfactor_dvd_core_modP`)
plus a nonzero-witness for the dividend (`hcore_mod_ne : core ≠ 0` at
site 1, `hcore_modP_iszero : core.isZero = false` at site 2), and
inline the same `cases hzero : factor.isZero` + `apply ext_coeff` +
`coeff_eq_zero_of_size_le` + `Hex.FpPoly.zero_mul` contradiction
chain (≈ 20 lines per site) to derive `factor.isZero = false`.

Site 1 (excerpt):

```lean
have hfactor_iszero :
    (@Hex.ZPoly.modP primeData.p primeData.bounds factor).isZero = false := by
  cases hzero : (@Hex.ZPoly.modP primeData.p primeData.bounds factor).isZero with
  | false => rfl
  | true =>
      exfalso
      have hfactor_zero :
          @Hex.ZPoly.modP primeData.p primeData.bounds factor = 0 := by
        apply Hex.DensePoly.ext_coeff
        intro n
        have hsize : (@Hex.ZPoly.modP primeData.p primeData.bounds factor).size = 0 := by
          change (@Hex.ZPoly.modP primeData.p primeData.bounds factor).coeffs.isEmpty = true at hzero
          simpa [Hex.DensePoly.size, Array.isEmpty_iff_size_eq_zero] using hzero
        rw [Hex.DensePoly.coeff_eq_zero_of_size_le
          (@Hex.ZPoly.modP primeData.p primeData.bounds factor) (by omega)]
        exact Hex.DensePoly.coeff_zero n
      rcases hfactor_dvd_core with ⟨q, hq⟩
      apply hcore_mod_ne
      rw [hfactor_zero, Hex.FpPoly.zero_mul] at hq
      exact hq
```

Site 2 has the same body but is itself the outer `cases` of a `by`
that proves `(modP factor).isZero = false` directly, and contradicts
the goal at the end via `Bool.noConfusion hcore_modP_iszero` instead
of `apply hcore_mod_ne … exact hq`.

These are the only two remaining inline `Hex.FpPoly p`-level
`dvd`+nonzero → `isZero = false` derivations in the BZMathlib tree
(see #5072's progress note, which flagged
`Basic.lean:1318` as the residual FpPoly-divisibility rewire after
the size-positivity-based isZero cleanup wave landed). The
`grep -n "Array.isEmpty_iff_size_eq_zero" HexBerlekampZassenhausMathlib/`
output reports exactly these two sites and no others.

The existing helper `Hex.isZero_false_of_ne_zero` at
`HexBerlekampZassenhaus/Basic.lean:486` handles the
`f ≠ 0 → f.isZero = false` direction for `Hex.FpPoly p`, but does
not cover the divisibility step. A small private helper added at the
`HexBerlekampZassenhausMathlib/Basic.lean` layer is the natural
container.
