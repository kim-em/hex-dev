## Deliverables

Audit the five just-landed `_of_bound` siblings (plus the
in-flight #4976 if landed by claim time) in
`HexBerlekampZassenhausMathlib/IntReductionMod.lean` for the three
consistency invariants listed below. Open follow-up issues for any
deviations, do not silently rewrite — this is a review pass.

1. **Docstring framing audit.** Each `_of_bound` sibling's
   docstring should open with the established "Abstract-bound
   variant of `<original>`:" framing (compare
   `centeredLift_scaledLiftedFactorProduct_eq_of_mignottePrecision_of_bound`
   at Basic.lean:1834 and
   `existsUnique_recoveringLiftedFactorSubset_of_henselSubsetCorrespondence_of_bound`
   at Basic.lean:1889 as canonical exemplars). The docstring should
   name (a) the concrete-form `hprecision` it replaces verbatim,
   (b) the abstract bound `B'` and the three premises that
   replace it (`hcore_lc_le`, `hvalid`, `hprecision`), and (c) the
   substrate forward call that becomes `_of_bound`. Each rewired
   original-now-wrapper's docstring should be unchanged from
   pre-rewiring (the public signature is preserved exactly).
   Report deviations with file:line and a one-line suggested fix.

2. **Parameter ordering audit.** The four-parameter abstract block
   must be inserted at the *same syntactic position* as the
   original `hprecision` binder, in the order
   `(B' : Nat) (hcore_lc_le ...) (hvalid ...) (hprecision : 2 * B' < d.p ^ d.k)`.
   No reordering relative to surrounding binders, no movement of
   the block to the head/tail of the signature. Confirm this for
   each of the five entries and report any deviation with
   file:line.

3. **Wrapper rewiring audit.** Each pre-existing original (now a
   thin wrapper) should:
   * Instantiate `B' := Hex.ZPoly.defaultFactorCoeffBound
     (Hex.normalizeForFactor f).squareFreeCore` (or the
     `core`-local equivalent, where the wrapper has a local
     `core` abbreviation in scope).
   * Discharge `hcore_lc_le` via
     `defaultFactorCoeffBound_valid core hcore_ne core
       hcore_dvd_self ((core.size - 1))` paired with
     `Hex.DensePoly.leadingCoeff_eq_coeff_last`.
   * Discharge `hvalid` via
     `defaultFactorCoeffBound_valid core hcore_ne` directly.
   * Forward the existing `hprecision` (now of shape
     `2 * defaultFactorCoeffBound core < d.p ^ d.k`) verbatim,
     with no `simp`/`exact?`-style cleanup that could obscure
     the bound type.

   Report any wrapper that diverges from this idiom (with
   file:line). Allowable variation: a wrapper proving
   `hcore_lc_le` via a *named* `have` block of the same shape
   instead of inline is acceptable, provided the substantive
   call remains `defaultFactorCoeffBound_valid` + the
   `leadingCoeff_eq_coeff_last` pairing.
