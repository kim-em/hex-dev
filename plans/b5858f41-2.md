## Current state

PR #4878 closed #4873 by landing the bottom three `_of_bound` siblings
of the A2 mignotte-precision recovery chain in
`HexBerlekampZassenhausMathlib/Basic.lean`, each accepting an abstract
bound `B' : Nat` plus a validity hypothesis
`hvalid : ∀ i, (factor.coeff i).natAbs ≤ B'` and a precision hypothesis
`2 * B' < d.p ^ d.k`, in place of the core-shape `2 *
defaultFactorCoeffBound core < d.p ^ d.k`.

`natDegree_toPolynomial_eq_sum_of_represents_of_primitive_pos_lc_core`
(`HexBerlekampZassenhausMathlib/Basic.lean:8218`) is the
primitive/positive-leading sibling of
`natDegree_toPolynomial_eq_sum_of_represents` (line 7545). Its proof
has two recovery-style steps:

* `_hrec_scaled` (line 8237) — calls
  `scaledRecombinationCandidate_eq_factor_of_recovery`. The binding is
  underscored — kept in the source only for API documentation; the
  result is never consumed by the rest of the proof.
* `hcenter` (line 8241) — calls
  `centeredLiftPoly_scaledLiftedFactorProduct_eq_factor_of_recovery`
  (the already-abstracted bottom-level lemma). This is the only
  recovery step the proof actually uses.

Because the `_hrec_scaled` line is dead weight, the abstract-bound
sibling can drop it entirely and route only through the bottom-level
`_of_bound` flavour landed by #4878. The result is independent of
both #4882 (scaled recovery-candidate `_of_bound` chain) and #4883
(unscaled recovery-candidate `_of_bound` chain) — matching the
independence pattern of #4890.

Contrast with the monic sibling
`natDegree_toPolynomial_eq_sum_of_represents` (line 7545): its proof
*actually* uses `recombinationCandidate_eq_factor_of_recovery_of_monic_core`
at line 7563, so its abstract-bound sibling is genuinely blocked on
#4883. Only this primitive variant happens to have an unused
recovery-candidate call site that lets us decouple it from #4882.
