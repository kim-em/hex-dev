## Current state

`HexPolyMathlib/Basic.lean` defines the Mathlib bridge between
`Hex.DensePoly R` and `Polynomial R` via `toPolynomial` /
`ofPolynomial`. The bridge already exposes the full set of
`toPolynomial_*` `@[simp]` wrappers (zero, C, one, add, neg, sub,
mul, ofPolynomial), but the symmetric `ofPolynomial_*` family is
not yet complete: `ofPolynomial_zero`, `ofPolynomial_one`,
`ofPolynomial_C`, `ofPolynomial_neg`, and `ofPolynomial_sub` exist
(landed via #5171 and #5244), while the binary ring operations
`ofPolynomial_add` and `ofPolynomial_mul` are missing.

Concretely, lines 247-254 of `HexPolyMathlib/Basic.lean`:

```lean
/-- `ofPolynomial` commutes with polynomial subtraction. -/
@[simp]
theorem ofPolynomial_sub [Ring R] [DecidableEq R] (p q : Polynomial R) :
    ofPolynomial (p - q) = ofPolynomial p - ofPolynomial q := by
  apply Hex.DensePoly.ext_coeff
  intro n
  rw [Hex.DensePoly.coeff_sub _ _ _ (by show (0 : R) - (0 : R) = 0; simp)]
  simp [coeff_ofPolynomial, Polynomial.coeff_sub]
```

The pair `ofPolynomial_add` / `ofPolynomial_mul` would complete the
binary-operation simp surface and let simp normalize
`ofPolynomial (p + q)` and `ofPolynomial (p * q)` automatically. The
matching `toPolynomial_add` (line 279) and `toPolynomial_mul`
(line 306) are already proven; the `equiv` ring-iso on line 340 also
provides `map_add` / `map_mul` on the inverse direction.

## Deliverables

Extend `HexPolyMathlib/Basic.lean`, immediately after
`ofPolynomial_sub` (line 254) and before `toPolynomial_zero`
(line 257), with two new `@[simp]` theorems:

```lean
/-- `ofPolynomial` commutes with polynomial addition. -/
@[simp]
theorem ofPolynomial_add [Semiring R] [DecidableEq R] (p q : Polynomial R) :
    ofPolynomial (p + q) = ofPolynomial p + ofPolynomial q := ...

/-- `ofPolynomial` commutes with polynomial multiplication. -/
@[simp]
theorem ofPolynomial_mul [CommRing R] [DecidableEq R] (p q : Polynomial R) :
    ofPolynomial (p * q) = ofPolynomial p * ofPolynomial q := ...
```

For `ofPolynomial_add`, mirror the `ofPolynomial_sub` proof shape:
extensionality via `Hex.DensePoly.ext_coeff`, then
`Hex.DensePoly.coeff_add` with the obvious `(0 : R) + (0 : R) = 0`
witness, then `simp [coeff_ofPolynomial, Polynomial.coeff_add]`.

For `ofPolynomial_mul`, the simplest path is via the existing
`equiv` ring-isomorphism (line 340): unfold `equiv.symm` to
`ofPolynomial`, then use `map_mul (equiv (R := R)).symm`. If that
shape is awkward, prove it by rewriting through
`toPolynomial_ofPolynomial` and `toPolynomial_mul`. `CommRing` is
acceptable here (it matches the `equiv` typeclass), but use the
weaker `Semiring` / `CommSemiring` if straightforward.

Use the proof shape that requires the weakest typeclass that
compiles; do not introduce avoidable strengthenings.
