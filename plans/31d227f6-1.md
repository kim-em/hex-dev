## Current state

`Hex.factorFastFactorsWithBound : ZPoly → Nat → Option (Array ZPoly)`
(`HexBerlekampZassenhaus/Basic.lean:5096`) dispatches across a
nested-`if` cascade with nine possible outcomes (constant-core, `B = 0`,
four `B = 1` sub-cases, three `B > 1` sub-cases) before producing
either `some (reassemblePolynomialFactors normalized _)` or `none`.

The slow path already has a complete dispatch classifier:

```lean
theorem factorSlowFactorsWithBound_branch (f : ZPoly) (B : Nat) :

    -- quadratic
    (∃ coreFactors, ... ∧ squareFreeCore.degree?.getD 0 ≠ 0 ∧
      quadraticIntegerRootFactors? squareFreeCore = some coreFactors) ∨
    -- exhaustive
    (factorSlowFactorsWithBound f B = ... ∧ squareFreeCore.degree?.getD 0 ≠ 0 ∧
      quadraticIntegerRootFactors? squareFreeCore = none)
```

(`HexBerlekampZassenhaus/Basic.lean:5043`, three-way disjunction).

There is **no companion classifier** for the fast path. The existing
per-branch entry-shape lemmas
(`factorWithBound_entry_mem_constant_branch_raw`,
`factorWithBound_entry_mem_small_mod_singleton_raw`,
`factorWithBound_entry_mem_quadratic_branch_raw`,
`factorWithBound_entry_mem_fast_core_success_raw`,
`factorWithBound_entry_mem_slow_quadratic_branch_raw`,
`factorWithBound_entry_mem_exhaustive_branch_raw`) each take their
branch-marker hypotheses as **explicit** inputs, so callers must either
inline the nested-`if` case-split or duplicate the dispatch reasoning.

The HO-1 capstone `factor_irreducible_of_nonUnit`
(`HexBerlekampZassenhausMathlib/Basic.lean:237`, #4170) will need this
case-split to dispatch each recorded entry to the right per-arm
umbrella. Hoisting it into a single reusable theorem mirrors the slow
side and lets the capstone proof remain a straight composition rather
than re-doing the nested dispatch by hand.
