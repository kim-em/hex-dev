## Current state

PR #4878 closed #4873 by landing the bottom three `_of_bound` siblings
of the A2 mignotte-precision recovery chain in
`HexBerlekampZassenhausMathlib/Basic.lean`. Subsequent PRs propagated
`_of_bound` upward through the recovery-candidate and
`representsIntegerFactorAtLift` layers; #4901 covers the
support-containment substrate at lines 3886 / 3914 / 3946 and #4902
covers the unscaled-side `exists_representingSubset_*` bridges at
lines 9488 / 9597.

The body of #4902 explicitly defers the **scaled-side**
`exists_representingSubset_of_mem_normalizedFactors_scaledRecombinationCandidate_of_primitive_pos_lc_core`
bridge at line 10488. The reason is that the scaled-side bridge's
proof calls
`zpoly_primitive_scaledRecombinationCandidate hcore_ne hcore_lc_pos
    hd_liftedFactor_monic hprecision T`
(line 10645), and `zpoly_primitive_scaledRecombinationCandidate`
(line 10453) is a **non-vestigial** precision consumer: its proof
uses `hprecision : 2 * Hex.ZPoly.defaultFactorCoeffBound core <
d.p ^ d.k` to call
`size_centeredLiftPoly_eq_of_pos_leadingCoeff_bound` at line 10519
(via an `hslp_lc_bound` derived from `defaultFactorCoeffBound_valid`).

`size_centeredLiftPoly_eq_of_pos_leadingCoeff_bound` already accepts
an abstract `B : Nat` together with
`(lc g).natAbs ≤ B` and `2 * B < m` (definition at line 8224, signature
`{g : Hex.ZPoly} {m B : Nat}`). So `zpoly_primitive_scaledRecombinationCandidate`
can be lifted to an `_of_bound` sibling by replacing
`defaultFactorCoeffBound_valid` derivation with an abstract bound on
`leadingCoeff core`.

The bound only applies to one coefficient — `Hex.DensePoly.leadingCoeff
core` — because that is the only coefficient `defaultFactorCoeffBound_valid`
is invoked at inside this proof (line 10502, `defaultFactorCoeffBound_valid
core hcore_ne core (dvd_refl core) (core.size - 1)`). So the abstract-
bound hypothesis is a single inequality on the leading coefficient,
not the universal `∀ i, (factor.coeff i).natAbs ≤ B'` shape used by
the factor-side `_of_bound` siblings.

This issue lands `zpoly_primitive_scaledRecombinationCandidate_of_bound`
as a single atomic PR. It unblocks the deferred scaled-side
`exists_representingSubset_*_of_bound` sibling, which a follow-on issue
will pick up after this lands.
