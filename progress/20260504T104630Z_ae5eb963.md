# HexPoly: divModArrayAux reconstruction from array steps (#1991)

## Accomplished

- Added `divModArrayAux_reconstruction` (private, CommRing) to
  `HexPoly/Euclid.lean`. The lemma is parameterized by a freshly
  chosen bound `B` per recursive call; under
  `q.size = qDegree + 1`, the cancellation hypothesis for
  `scaleLead`, the standard rem/quot sparsity bounds, and a size
  match `rem.size ≤ qDegree + quot.size`, it shows
  `ofCoeffs qr.1 * ofCoeffs q + ofCoeffs qr.2
    = ofCoeffs quot * ofCoeffs q + ofCoeffs rem`.
  The proof uses `ofCoeffs_set!_eq_add_monomial` for the quotient
  step, `ofCoeffs_subtractScaledShift_eq_sub_monomial_mul` for the
  remainder step, and `divMod_reconstruction_step` for the
  algebraic preservation. Strict descent of `shift = rd - qDegree`
  through the recursion lets `B` decrease to `shift` for the IH,
  keeping `quot[shift] = 0` at each write.
- Added `divModArray_reconstruction` (theorem, public) which
  derives `(divModArray p q scaleLead).1 * q
    + (divModArray p q scaleLead).2 = p`
  from the recursive lemma, handling the `q.isZero` shortcut via
  `zero_mul`/`zero_add` and otherwise invoking the inductive lemma
  with `B := p.size` on the initial all-zero `quot` of size
  `p.size - qDegree`.
- `lake build` green; `python3 scripts/check_dag.py` clean;
  `git diff --check` clean; no new `sorry`/`axiom`/`native_decide`
  /`TODO`/`FIXME` placeholders.
