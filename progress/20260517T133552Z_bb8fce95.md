## Proof shape

Induct on `k`:

- `k = 0`: `ZMod (p^0)` is the zero ring, so both equations hold
  vacuously by `Subsingleton.elim`.  The proof keeps `0 < k` in the
  signature for downstream callers but does not need it.

- `k + 1`: derive level-`k` versions of `hprod`/`hprod'` via
  `polynomial_map_zmod_pow_succ_to_pow`, apply IH to get `g.map φ_k =
  g'.map φ_k` and `h.map φ_k = h'.map φ_k`.  Extract `A`, `B` with
  `g - g' = C (p^k) * A` and `h - h' = C (p^k) * B`.  Expand
  `g*h - g'*h' = C (p^k) * (A*h + g'*B)`; the level-`p^(k+1)` LHS
  vanishes, so each integer coefficient of `A*h + g'*B` is divisible
  by `p`.  Branch on `g.natDegree`: if zero, `g = g' = 1` and both
  conclusions follow directly from `hprod`/`hprod'`.  Otherwise apply
  `isCoprime_cancel_of_natDegree_lt` to get `A.map φ_p = B.map φ_p =
  0`, then lift back to level `p^(k+1)` via the helper.

`hh1` is redundant given the rest of the API (the cancellation step
uses `hg1` to substitute `g'` for `g` mod `p`; the symmetric form would
have used `hh1` instead).  Kept in the signature with `_hh1` to
preserve the issue-specified API; this matches the existing convention
for `_hk` in `coprime_mod_p_lifts`.
