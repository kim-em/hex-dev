### Diagnostic finding: formula gap was already closed by PR #2823

The `reports/hex-lll-performance.md` §"Verdicts" subsection cites
`reports/bench-results/hex-lll-e211854d1435.json` as the latest
scientific artifact and reports five inconclusive parametric
verdicts:

- `runSizeReduceChecksum`
- `runGramSchmidtCoeffChecksum`
- `runFirstShortVectorHarshCubicChecksum`
- `runOfBasisHarshCubicChecksum`
- `runFirstShortVectorRandomBoundedChecksum`

That artifact was measured at commit `e211854d` (the perf-fuse PR
[#2802](https://github.com/kim-em/hex/pull/2802)) with the formulas
that existed at that commit:

```
firstShortVectorRandomBoundedComplexity n := n ^ 4
firstShortVectorHarshCubicComplexity n    := n ^ 5
```

The verdicts in the JSON were computed against those formulas.
The `ratios` field equals `per_call_nanos / n^4` (resp. `/ n^5`),
which matches the recorded `[(30, 6.890), (60, 3.649), (120, 2.166),
(240, 1.817)]` for RandomBounded and `[(15, 1.541), (30, 0.924),
(45, 0.963)]` for HarshCubic. Both ratio series violate the slope
tolerance, hence "inconclusive".

PR [#2823 "Repair HexLLL phase4 benchmark verdicts"](https://github.com/kim-em/hex/pull/2823)
landed immediately after `e211854d` and (a) updated the formulas to:

```
firstShortVectorRandomBoundedComplexity n := n ^ 3 * Nat.log2 (n + 1)
firstShortVectorHarshCubicComplexity n    := n ^ 4 * (Nat.log2 (n + 1)) ^ 3
```

and (b) committed a NEW artifact `reports/bench-results/hex-lll-885431e.json`
(commit `885431e`) showing all five formerly-inconclusive registrations as
`consistent_with_declared_complexity` with valid slopes inside tolerance:

| Function | Verdict | Slope | (c_min, c_max) |
|---|---|---|---|
| `runSizeReduceChecksum` | consistent | (narrow-range) | (6.20, 7.00) |
| `runGramSchmidtCoeffChecksum` | consistent | 0.068 | (1080.0, 1184.6) |
| `runFirstShortVectorHarshCubicChecksum` | consistent | -0.013 | (0.349, 0.430) |
| `runOfBasisHarshCubicChecksum` | consistent | (narrow-range) | (7.62, 10.01) |
| `runFirstShortVectorRandomBoundedChecksum` | consistent | 0.103 | (42.52, 62.62) |

So the formula-vs-implementation gap that produced the inconclusive
verdicts has been closed in code since PR #2823; the same code is on
current `main`.
