## Key decisions / patterns

- **Run Mathlib `#lint` in-file, not via a scratch import.** `#lint` only
  checks declarations defined in the file where it runs; an
  `import HexPolyZMathlib.RobinsonForm` scratch file would lint nothing.
  Appending `#lint` to the target file (then force-deleting its oleans to
  defeat the build cache) is the way to actually exercise `docBlame` et al.
  on the library's own decls. This is stronger than the project's usual
  "clean recompile = Lean built-in linters" interpretation and is worth
  doing whenever the library imports Mathlib.
- **Trivial public unfolding/simp lemmas got one-line docstrings, not
  paragraphs.** `robinsonFactor_of_norm_le`, `schurRootPath_zero/one`,
  `robinsonForm_zero` characterise the defs and are public API a caller
  can rely on, so a brief `/--` is appropriate (not "padding"); the two
  genuinely-private helpers got short invariant-stating docstrings.
- **Stale-worktree hygiene (recurring).** The reused worktree again had
  uncommitted deletions stripping the dead-code-scan guidance from
  `.claude/commands/review.md` and the `agent-worker-flow` skill (no PR,
  no commit) — the same corruption the 2026-06-20T21:15 session flagged.
