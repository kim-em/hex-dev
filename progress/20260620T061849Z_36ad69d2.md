# HexPolyFp SquareFree dead-code removal (#8199)

## Accomplished

Removed **105 genuinely-dead private declarations** from
`HexPolyFp/SquareFree.lean` (Phase 6 exit criterion: no dead/unreferenced
declarations). File shrank 10997 → 8118 lines (2879 net lines removed).

Method: reachability scan with a **word-boundary, prime-aware** regex
over all `.lean` files, flagging private decls whose bare name appears
only at its own definition. None of the 427 private decls carry
`@[simp]`/`@[grind]`/`instance`, so the attribute-root trap did not
apply. Removal was an iterative **delete → rebuild → rescan fixpoint**
(9 rounds) to catch cascading orphans; every round kept
`lake build HexPolyFp` green. Reached a dry fixpoint at round 10.

Two regex subtleties mattered:
- A dot-*excluding* leading boundary is correct here because the targets
  are `private` (file-local, never referenced qualified across files). A
  dot-*including* boundary over-counts via cross-namespace name
  collisions — e.g. `HexPolyFp`'s private `derivative_pow_succ` collides
  with Mathlib's `Polynomial.derivative_pow_succ`, which made it look
  live when it is dead.
- The issue's spot-check claimed `yunFactorsContribution_reconstruct` was
  live with "8 occurrences", but that was a naive substring grep
  conflating it with the distinct `_core`/`_done`/`_tail` siblings. The
  bare name was dead; removing it cascaded the siblings dead too.
