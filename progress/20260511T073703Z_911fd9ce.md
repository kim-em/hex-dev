# Phase 6 review: HexArith/Nat/ModArith.lean (#3289)

## Accomplished

Reviewed `HexArith/Nat/ModArith.lean` against `SPEC/Libraries/hex-arith.md`
(Layer 2) and `PLAN/Phase6.md`. Of the five lemmas in the file, only
`Nat.coprime_pow_two_of_odd` had a downstream caller (a single use in
`HexArith/Montgomery/Context.lean:225` inside `cancel_word_mod_of_lt`). The
remaining four were dead weight:

- `Nat.div_mul_of_dvd` — one-line wrapper around `Nat.div_mul_cancel`
  (already in core), no callers.
- `Nat.mod_add_div_eq` — `simpa [Nat.mul_comm]` of `Nat.mod_add_div`
  (already in core), no callers; downstream callers use `Nat.mod_add_div`
  directly.
- `Nat.coprime_twoPow_right` — swapped-argument form of
  `coprime_pow_two_of_odd`, no callers.
- `Nat.twoPow_dvd_mul_add_one_of_mod_eq_pred` — one-line wrapper over
  `Nat.mod_eq_sub_iff`, no callers; `mod_eq_sub_iff` is not used anywhere
  else in the project either.

Phase 6 exit criteria explicitly require "no dead or unreferenced
declarations", so deleted all four. Each is a one-line proof and can be
re-introduced trivially if a future Montgomery/Barrett refactor needs it.
Updated the file docstring to match the new (single-lemma) content.
