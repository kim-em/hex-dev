# 2026-05-15 07:51 UTC — feature session, issue #3660 (HO-17)

## Accomplished

- Added `scripts/oracle/lll_fpylll_bench_driver.py`: a persistent
  subprocess driver that imports `fpylll` once at startup and loops
  on stdin. Each request is one line of Haskell-syntax `[[Integer]]`
  matrix (matching `Hex.LLLBench.matrixHaskell`); each reply is one
  line containing the first-row checksum
  (`Hex.LLLBench.intVectorChecksum`) of `fpylll.LLL.reduction`'s
  output at `δ = 0.75`. Malformed requests come back as
  `ERROR: ...` lines and never terminate the driver.
- Rewrote `Hex.LLLBench.runFpylllFirstShortVectorChecksum` to send
  the same `matrixHaskell` request through the persistent driver
  via a new `fpylllChildRef`-cached `PersistentComparator`. The
  parser treats non-integer or `ERROR:` replies as driver faults
  and retries once after re-spawning. Driver path and Python
  interpreter are overridable via `HEX_LLL_FPYLLL_BENCH_DRIVER`
  and `HEX_LLL_FPYLLL_BENCH_PYTHON`.
- Removed the now-unused `firstShortVectorMatrixInput` whitespace
  formatter; the persistent path uses `matrixHaskell` (shared with
  the Isabelle wiring).
- Refreshed the module docstring to document the joint protocol,
  the per-driver `IO.Ref` lifetimes, and the driver-path
  overrides. Added measured per-call figures.
- Measured per-call overhead on the audit host (CPython 3 + fpylll
  2.5.6):
  - Legacy per-call shell-out shape: ~116 ms median per call
    (CPython start + `import fpylll` + one reduction).
  - Persistent driver in steady state: ~34 µs median per call
    (5 trials × 1000 trivial `[[1,0],[0,1]]` requests, after one
    warm-up).
  - Order-of-magnitude speedup: ~3 400× in steady state.
- `lake build HexLLL hexlll_bench`, `python3 scripts/check_dag.py`,
  and `git diff --check` all pass. Each of the three fixed fpylll
  comparator benchmarks (`runFpylllFirstShortVectorBZRecombination`,
  `runFpylllFirstShortVectorHarshCubic15`, and
  `runFpylllFirstShortVectorRandomBounded30`) runs end-to-end with
  `expectedHash` matching after the rewrite.
