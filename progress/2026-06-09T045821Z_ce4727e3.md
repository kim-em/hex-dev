# D5b: dispatch implementation of public lll

## Accomplished

- Rewrote `Hex.lll` in `HexLLL/Basic.lean` to gate on
  `LLLProvider.providerAvailable ()` before any input marshalling; if
  the provider is present, marshals the basis, calls `providerReduce`
  at `η = 0.55` (≈ 11/20), reshapes the result, and runs `certCheck`
  at exact `(δ, 11/20)`. Accepts the certified `B'` on a passing check,
  otherwise falls back to `lllNative`. Diagnostic outcomes
  (`absent / providerError / rejected / accepted`) are bumped from pure
  code via a `@[implemented_by]`/`unsafeBaseIO` companion
  (`withRecordOutcome`).
- New helpers in `LLLProvider`: `withRecordOutcome` (pure-typed
  diagnostic side-effect), `ratToFloat`, `matrixToEntries`,
  `matrixFromArray`, the bundled `CertifiedTriple`, the shape/cert
  pipeline `certifyFlat`, and the gate→marshal→certify orchestrator
  `dispatch`. The Mathlib-free extraction lemma
  `dispatch_some_certCheck` produces the witnesses fed into
  `HexLLLMathlib.certCheck_sound`.
- Property-level extraction `Hex.dispatch_some_property` added in
  `HexLLLMathlib/Independent.lean`. The four headline theorems
  (`lll_isLLLReduced`, `lll_memLattice_iff`, `lll_independent`,
  `lll_mem_latticeSubmodule_iff`) now case-split on
  `LLLProvider.dispatch b δ`: the `none` branch runs the original
  native-fallback proofs, the `some` branch reads the conclusion off
  `dispatch_some_property`. `lll_short_vector` and
  `lll_first_row_norm_sq_le_unconditional` carry through unchanged
  because they consume the headline theorems abstractly.
