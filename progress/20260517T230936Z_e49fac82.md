### #4639 — HO-1 substrate exhaustive wrapper relaxation

Claimed and audited. The relaxation as written is currently
undeliverable: the public wrapper exposed by #4648
(`recombinationSearchModAux_some_factor_associated_of_liftedFactorSubsetPartition_of_primitive_pos_lc_core`)
concludes about `Hex.scaledRecombinationSearchModAux`, while the
existing membership wrapper
`exhaustiveCoreFactorsWithBound_mem_of_recombinationSearchMod_some`
and the executable `Hex.exhaustiveCoreFactorsWithBound`
(`HexBerlekampZassenhaus/Basic.lean:5363`, calling `recombineExhaustive`
at line 5010) are bound to the unscaled `Hex.recombinationSearchMod`.
There is no unscaled primitive variant — per #4647 the unscaled
candidate shape is rejected as false for primitive non-monic cores. So
the coverage theorem about the existing unscaled executable cannot be
relaxed to primitive + positive-leading cores; for reducible primitive
non-monic input the executable falls back to `#[core]` and coverage
fails.

Skipped with a detailed analysis comment proposing a successor
decomposition: (1) executable-side bridge switching
`recombineExhaustive` / `Hex.exhaustiveCoreFactorsWithBound` to the
scaled search; (2) a scaled membership wrapper in
`HexBerlekampZassenhausMathlib/Basic.lean`; (3) only then the three
wrapper relaxations originally scoped here.

See https://github.com/kim-em/hex/issues/4639#issuecomment-4472787565.
