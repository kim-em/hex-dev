## Accomplished

Discharged the `j = 0` branch of the residual `j ≤ i` obligation in
`getArrayEntry_scaledCoeffRowsSchur_eq` in `HexGramSchmidt/Int.lean`, and
added the structural frame lemmas D2/D3 will need.

Schur-side helpers (added near the existing
`getArrayEntry_schurRowLoop_upper`):

- `getArrayEntry_schurColumnLoop_row_ne`: column loop preserves cells in
  any row other than the loop's active row.
- `getArrayEntry_schurRowLoop_row_not_mem`: row loop preserves cells in
  rows outside `rowList`. This is the row-completion frame lemma.
- `getArrayEntry_schurColumnLoop_col_zero_preserve`: column loop preserves
  `(row, 0)` when `0 ∉ cols`.
- `getArrayEntry_schurColumnLoop_col_zero`: when `0 ∈ cols`, column loop
  writes `gram[row][0]` into `(row, 0)`.
- `schurColumnLoop_size` and `schurColumnLoop_rows_size`: column-loop
  size preservation, used to thread bounds through the row-loop dispatch.
- `getArrayEntry_schurRowLoop_col_zero`: row-loop dispatch for the
  `(i, 0)` boundary.
- `schurSigma_congr_of_cell_eq`: σ-chain dependency frame lemma. The
  value of `schurSigma rows i j` depends only on `rows[k][k]` for `k < j`,
  `rows[i][p]` for `p < j`, and `rows[j][p]` for `p < j`.
- `getArrayEntry_scaledCoeffRowsSchur_col_zero`: top-level `j = 0`
  boundary fact `(scaledCoeffRowsSchur b)[i][0] = (gramRows b)[i][0]`.

Bareiss-side helpers (added after
`getArrayEntry_scaledCoeffArrayLoop_current_col_written`, where the
referenced branch lemmas are available):

- `scaledCoeffArrayLoop_coeffs_size`: outer-array length is preserved
  through the entire array loop.
- `getArrayEntry_scaledCoeffRows_col_zero`: counterpart fact
  `(scaledCoeffRows b)[i][0] = (gramRows b)[i][0]` for the array path.

The main theorem `getArrayEntry_scaledCoeffRowsSchur_eq` now case-splits
`by_cases hij : i < j` and then `by_cases hj : j = 0`, dispatching the
`j = 0` slice via the two new top-level col-zero lemmas. The residual
`sorry` now covers only `0 < j ≤ i`.
