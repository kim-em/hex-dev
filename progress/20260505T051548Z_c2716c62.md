## Accomplished

Discharged issue #2279 (polynomial-side geometric-series divisibility for
the Rabin bridge): added the public theorems

- `FpPoly.coeff_monomial_mul`: `(monomial k c * g).coeff n = if n < k
  then 0 else c * g.coeff (n - k)`. Built on the existing
  `mulCoeffSum_eq_degree_bound` foldl decomposition and a careful
  three-case analysis (`k < m`, `k = m`, `k > m`) of the inductive step.
- `FpPoly.monomial_mul_monomial`: `monomial m 1 * monomial n 1 =
  monomial (m + n) 1`. Direct from `coeff_monomial_mul` plus
  `coeff_monomial`.
- `FpPoly.monomial_zero_one_eq_one`: `monomial 0 1 = 1`. Plumbing for
  the base case of `linearPow_monomial`.
- `FpPoly.linearPow` (public def) plus `linearPow_succ`,
  `linearPow_succ_left`, `linearPow_add`, `linearPow_monomial`,
  `linearPow_monomial_one`. The naming `linearPow` (rather than just
  `pow`) avoids collision with the existing private `pow` square-and-
  multiply definition in `HexPolyFp/SquareFree.lean`.
- `FpPoly.sub_one_dvd_linearPow_sub_one`: `(Y - 1) ∣ (linearPow Y j - 1)`
  by induction on `j`. The inductive step exhibits `q * Y + 1` as the
  divisor witness, using `left_distrib`, `mul_one`, `mul_assoc`, the new
  private `sub_one_mul_eq` (which leverages `DensePoly.neg_mul_right_poly`
  for `(-1) * Y = -Y`), and a final coefficient-wise grind over
  `DensePoly.coeff_sub`/`coeff_add` to discharge the `A - 1 = A - Y +
  (Y - 1)` regrouping.
- `FpPoly.monomial_sub_one_dvd_of_dvd`: the issue's main deliverable.
  Decomposes `l = k * j` from `k ∣ l`, rewrites `monomial (k * j) 1` as
  `linearPow (monomial k 1) j` via `linearPow_monomial`, then applies
  the geometric-series divisibility.

## Implementation note

`FpPoly` does not carry a `Lean.Grind.CommRing` instance, so the
polynomial algebra had to go through the named theorems
(`left_distrib`, `mul_assoc`, `mul_one`, `sub_eq_add_neg`, …). The
private helper `sub_one_mul_eq : (P - 1) * Y = P * Y - Y` extracts the
one identity that the geometric step needs but isn't already in the
file; it bottoms out in `DensePoly.neg_mul_right_poly` (which requires
`Lean.Grind.CommRing (ZMod64 p)`, available via `HexModArith.Ring`).

The naming of `linearPow` parallels the private `powLinear` already
inside `HexPolyFp/Frobenius.lean`. The optional deliverable 3
(exposing `Frobenius.powLinear`) was not pursued: providing
`linearPow` in `Basic.lean` keeps the divisibility chain self-contained
without disturbing `Frobenius.lean`'s repeated-squaring infrastructure.
