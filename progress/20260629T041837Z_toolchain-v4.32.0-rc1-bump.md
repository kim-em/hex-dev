- `lean-toolchain` вЖТ `leanprover/lean4:v4.32.0-rc1`; `lakefile.lean` verso and

- **All `.lean` source in the monorepo now compiles under v4.32.0-rc1.** Roughly
  95 fix sites across ~25 files. The dominant cause is the Lean 4.31 change
  (carried into 4.32): the final goal/term match in `simpa using e` now uses
  *reducible* transparency, so it no longer unfolds plain `def`s. Pure
  defeq-bridge `simpa [...] using e` calls were rewritten to `exact e`; where
  real rewriting plus a defeq residual was involved, the rewrite was split out
  (`simp only [...] at h; exact h`, or an explicit `rw`/type-ascription).
  Representative defeq pairs bridged: `a*b`вЖФ`a.mul b`, `aвБї¬є`вЖФ`a.inv`,

- Non-`simpa` v4.32 fixes: `getElem!_def` now expands to a literal `match` (added

  so a few `dsimp at h` lines became no-ops (replaced by a typed `have`);
  `smul_apply` is newly ambiguous (qualified to `Matrix.smul_apply` in
  `HexDeterminantMathlib/DesnanotJacobi`); `simp` now leaves `if True`/`if c`
  unreduced in a couple of Matrix proofs (added `if_true`/`if_neg`).

## Current frontier вАФ two non-source blockers remain

1. **`lean-bench` dependency does not build on v4.32.** `LeanBench/RunEnv.lean:220`
   calls `Std.Time.PlainDateTime.ofTimestampAssumingUTC`, which was REMOVED in
   v4.32's `Std.Time` (the TimestampвЖФPlainDateTime API was restructured; no
   drop-in rename). `lean-bench` is pinned `@ master` and is a separate repo
   (`kim-em/lean-bench`); every `*_bench` exe depends on it, so the CI bench
   build cannot complete until lean-bench is fixed upstream and re-pinned.

2. **Lake build cycle with `precompileModules` on v4.32.** A bare `lake build`
   (default target `HexManual`) and any precompiled-lib-dependent exe (e.g.
   `hexgfqfield_bench`) hit:
   `batteries/Batteries:shared вЖТ batteries/BatteriesRecycling:shared вЖТ
   Hex/+HexMatrix.Basic:setup вЖТ вА¶ вЖТ вА¶:exe/HexManual:default вЖТ
   batteries/Batteries:shared`.
   The spurious edge `BatteriesRecycling:shared вЖТ HexMatrix.Basic:setup` is a
   v4.32 Lake regression in how `:shared` deps are computed for precompiled
   modules under the new module system (the tree is green on v4.30 with the same
   lakefile). The CI library targets themselves compile; the cycle bites the
   bench exes and the verso manual. A candidate workaround is dropping
   `precompileModules := true` (8 libs in `lakefile.lean`), which would not change
   compiled-exe runtime but is a SPEC/perf call вАФ left for review, not applied.
