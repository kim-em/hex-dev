## Accomplished

- Added `clmulCoeffAt_above_degree_add_eq_false` (private) in
  `HexGF2/Multiply.lean`: for any pair of source words `(i, j)` and any global
  bit index `n > dp + dq`, the source-pair contribution
  `clmulCoeffAt (i + j) p.words[i]! q.words[j]! n` vanishes. The proof
  splits on the three `clmulCoeffAt_sourcePairCoeff` cases (`n / 64 = i + j`,
  `n / 64 = i + j + 1`, otherwise) and then, for any `(a, b)` with
  `a + b + 64 * (i + j)` matching `n`, uses `omega` to derive
  `dp < 64 * i + a` or `dq < 64 * j + b`, killing the bit via
  `wordBitAt_getElem!_eq_false_of_degree?_lt`.
- Added `coeffWords_mulWords_above_degree_add_eq_false` (private): combines
  the per-pair vanishing with `coeffWords_mulWords_contrib`,
  `xorBoolList_map_xorBoolList`, and `xorBoolList_range_false` to conclude
  the raw product coefficient at `n > dp + dq` is `false`.
- Added `coeff_mul_eq_false_of_degree_add_lt` (public): the `coeff_mul`
  unwrap of the above for downstream callers.
- Added `degree?_mul_of_degree?_eq_some` (public): the headline theorem
  required by #1916 — `(p * q).degree? = some (dp + dq)` whenever
  `p.degree? = some dp` and `q.degree? = some dq`. Proved by
  `degree?_eq_some_of_coeff_eq_true_of_forall_gt_false` with
  `coeff_mul_degree_add_of_degree?_eq_some` (#1940) supplying the witness
  and `coeff_mul_eq_false_of_degree_add_lt` supplying the upper-tail vanish.
