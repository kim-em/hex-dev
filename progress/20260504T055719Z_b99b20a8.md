## Accomplished

- Closed #2019 by bumping `HexGfqRing/Conformance.lean` from the
  degree-2 modulus `x^2 + x + 1` to the degree-4 modulus `x^4 + 2`
  over `F_5`, matching the modulus already used by `HexGfqField`
  conformance after #2030.
- Reduction rule: `x^4 ≡ -2 ≡ 3 (mod m, mod 5)`. Recomputed every
  `#guard` literal by hand from this rule:
  - `a = 2 + 3x` (typical, deg 1)
  - `b = 4 + x + x^3` (edge, deg 3 = max reduced degree)
  - `x` (generator, unchanged)
  - `x4Raw / x4Mod` raw polynomial `x^4` and its reduction to `3`
  - `x5 = q [0,…,1]` (length 6) reduces to `3x` exercising
    quotient-by-quotient on degrees above the modulus
- Highlights:
  - `a + b = 1 + 4x + x^3 → [1, 4, 0, 1]`
  - `a * b = 2 + 4x + 3x^2 + 2x^3 → [2, 4, 3, 2]` (boundary case
    where the `3x^4` term reduces to `9 ≡ 4` shifted into the
    constant)
  - `x^4 → [3]`, `x^5 → [0, 3]` exercise the reduction at and above
    the modulus boundary
  - Subtraction `b - x4Mod = 1 + x + x^3 → [1, 1, 0, 1]` exercises
    the maximal-degree reduced representative
  - Negation `-b → [1, 4, 0, 4]` covers the deg-3 edge case
- Verified deliberate corruption of `reprNats x4Mod = [3]` to `[2]`
  trips the build, then reverted.
- Elaboration time: 697ms warm, ~1.0s cold — well under the 2s
  soft budget.
