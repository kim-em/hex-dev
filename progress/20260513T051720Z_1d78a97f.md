# 20260513T051720Z — HO-41 `.lake/build` GitHub Actions cache

## Accomplished

- Added `actions/cache@v4` step to both `.github/workflows/ci.yml`
  (`build` job) and `.github/workflows/conformance.yml`
  (`conformance` job), placed after `Configure Lean shared library
  path` and before the first `lake build`, per issue #3704.
- Both cache steps share the same shape:
  - `path` covers `.lake/build/lib/lean/Hex*` and `.lake/build/ir/Hex*`
    (Hex modules only, not Mathlib — Mathlib `.olean` continues to come
    from `lake exe cache get`).
  - `key: lake-build-${{ github.workflow }}-${{ github.run_id }}` so
    every run saves a fresh snapshot.
  - `restore-keys: lake-build-${{ github.workflow }}-` matches any
    earlier run of the same workflow.
