## Current frontier

#2247's full deliverable cannot be stated as written: it asks for
`d ^ (n + 1) ∣ repeated`, but `DensePoly Rat` has no `HPow _ Nat _`
instance (no `Monoid` instance either — the codebase only has `Mul`,
`One`, etc. plus `Lean.Grind.CommRing` for proofs). Test
`example (d : DensePoly Rat) : DensePoly Rat := d ^ 2` fails with
`failed to synthesize HPow (DensePoly Rat) Nat (DensePoly Rat)`.

The helper landed here is the algebraic core needed by the induction
either way — what's missing is a way to phrase / iterate the divisibility
chain.

## Next step

A planner needs to re-shape #2247 in one of these directions before the
next worker tries to land it:

1. **Define a local pow.** Add a private recursive
   `ratPolyPow d n` (`Nat`-iteration of `(· * d)` over `1`) to
   `HexPolyZ/Basic.lean` and state the inductive lemma against it.
2. **Reformulate without pow.** State the iteration via existential
   witnesses, e.g.
   ```
   ∀ n, ∃ k, repeatedRat = d * (Nat.rec d (fun _ acc => d * acc) n) * k
   ```
   or a chained "extension" lemma:
   `(c : DensePoly Rat) → c ∣ repeatedRat → d * c ∣ repeatedRat`.
   The latter doesn't quite hold without additional invariants on `c`,
   so option (1) is probably cleaner.
3. **Add a general `Monoid (DensePoly R)` instance** in `HexPoly` (out
   of scope for HexPolyZ).

For #2248 (the size argument), the same Pow gap applies and a parallel
re-shape is needed.
