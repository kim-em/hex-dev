# Center BHKS cldCoeffs cut at ambient p^a (#6220)

## Accomplished

- Centered the BHKS `psiCut` input at the ambient modulus `p^a` before
  taking the lower `p^ell_j` cut. The exact `f * g.derivative / g`
  coefficient is now reconstructible from the executable
  `cldQuotientMod` representative whenever `2 * bhksCoeffBound f j < p^a`,
  closing the #6217 counterexample where `f = x^2 - 5x + 6`, `g = x - 2`,
  `p = 5`, `a = 6` produced `cldCoeffs[0] = 125` against
  `bhksCoeffBound f 0 = 16`.
- Added a `#guard` regression for the counterexample: with the centered
  cut, `(cldCoeffs cldGuardF 5 6 cldGuardG).getD 0 0 = 0`.
- Exposed three Mathlib-free helpers in `HexBerlekampZassenhaus/Basic`:
  - `Hex.centeredModNat_emod_self`
  - `Hex.centeredResiduePow_eq_of_natAbs_le`
  - `Hex.psiCut_eq_zero_of_natAbs_le` plus the absolute-value form
    `Hex.abs_psiCut_le_of_natAbs_le`
- Added `Hex.cldCoeffs_getD_of_lt` exposing the in-range entry as a
  `psiCut` expression.
- Added the bridge theorem
  `Hex.BHKS.abs_cldCoeffs_le_bhksCoeffBound (f g) (p a j) (y) ...`
  packaging the BHKS Lemma 5.1 column bound under the Hensel precision
  hypothesis `2 * bhksCoeffBound f j < p^a` and `2 ≤ p`. Out-of-range
  `j` is handled by the `Array.getD` default.
- Removed `private` from `Hex.cldQuotientMod` so the bridge layer can
  state the congruence between the executable quotient and the exact
  integer CLD coefficient.
