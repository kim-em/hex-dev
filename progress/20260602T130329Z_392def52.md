# Session 392def52 — eliminate redundant divisibility check in exactDiv

Issue #6334. One-shot dispatch on `directive`.

## Accomplished

- Rebound `Hex.Matrix.exactDiv` to `lean_int_div_exact` via
  `@[extern]`, removing the upfront `Int.decidableDvd` mpz_mod that
  fired on every Bareiss update. Generated C for `stepMatrix` /
  `stepArray` now calls `lean_int_div_exact` directly with no
  divisibility branch (verified by inspecting
  `.lake/build/ir/HexMatrix/Bareiss.c`).
- Set `precompileModules := true` on `lean_lib HexMatrix` per
  SPEC/design-principles.md §Lakefile: once a Mathlib-free library
  exports an `@[extern]` symbol, downstream `#guard` callers need the
  precompiled dylib loaded by the elaborator (otherwise:
  "Could not find native implementation of external declaration
  'Hex.Matrix.exactDiv'").
- Re-benched the Lean harsh-cubic ladder on carica (`hexlll_bench
  run --repeats 3`) and refreshed
  `reports/bench-results/hex-lll-harsh-cubic-extended-warmupfix.json`
  (Lean rows only; Isabelle / fpylll untouched). Regenerated
  `reports/figures/hex-lll-comparator-harsh-cubic.svg`.
