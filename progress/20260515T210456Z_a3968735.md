# HexBZ Mathlib: matching predicate extensions for arbitrary J

Issue #4311 (sub-issue split from #4301 during this session): adds
arbitrary-J generalisations of the lifted-factor matching predicate
infrastructure, unblocking the main recursive coverage proof.

## Accomplished

In `HexBerlekampZassenhausMathlib/Basic.lean`, added the following lemmas
after the existing universe-level matching block (~line 2008):

- `finRange_filter_eq_sort` (private): the order-preserving filter of
  `List.finRange n` by membership in a Finset equals the sorted list of
  that Finset. Proven via `List.Perm.eq_of_sortedLE`, using
  `sortedLT_finRange` and `Pairwise.filter`.
- `finRange_filter_head?_eq_min'` (private): the `head?` of that filter
  for a non-empty Finset `J` is `some (J.min' hne)`. Follows from
  `Finset.sorted_zero_eq_min'`.
- `LiftedFactorListMatches.head?_eq_liftedFactor_min'` (public): the
  head of a non-empty matching state's `localFactors` is the lifted
  factor at `J.min'`.
- `finRange_filter_mem_and_mem_eq_of_subset`,
  `finRange_filter_mem_and_not_mem_eq_sdiff_of_subset` (private):
  filter-composition identities used to relate selected/rejected
  partitions of the matching state to `liftedSubsetSelectedList`.
- `LiftedFactorListMatches.sdiff_of_subset` (public): generalised
  matching transition — for any `J, S`,
  `LiftedFactorListMatches d (J \ S) (liftedSubsetSelectedList d (J \ S))`.
- `liftedSubsetSplit_mem_subsetSplitsWithFirst_of_matches` (public): the
  arbitrary-`J` generalisation of `liftedSubsetSplit_mem_subsetSplitsWithFirst`.
  Given a matching `LiftedFactorListMatches d J localFactors` and any
  `S ⊆ J` containing `J.min'`, the partition
  `(liftedSubsetSelectedList d S, liftedSubsetSelectedList d (J \ S))`
  lies in `Hex.subsetSplitsWithFirst localFactors`.

The proof of the partition lemma decomposes `localFactors = head :: tail`
via the head identity, builds the mask `bs = xs.map (· ∈ S)`, identifies
the selected/rejected filterMaps with the desired `liftedSubsetSelectedList`s
via the filter-composition identities, and applies the existing
`subsetSplitsWithFirst_zip_filterMap_partition`.
