# 2026-05-04T05:40Z — feature session 99d1a044

Issue: #1995 (HexHensel cypari2 factorpadic oracle).

## Accomplished

- Added `HexHensel/EmitFixtures.lean` — emits JSONL records for three
  multifactor Hensel-lift fixtures: `multifactor/typical` (p=5, k=4,
  three monic linear factors with non-trivial lifts to mod 5^4),
  `multifactor/mixed` (p=7, k=2, mixed-degree factor set including the
  irreducible quadratic `x²+1` mod 7), `multifactor/adversarial`
  (p=5, k=8, four monic linear factors exercising the quadratic
  doubling loop deeper).
- Added `scripts/oracle/hensel_pari.py` — cypari2 driver that calls
  `factorpadic(target, p, k)` for each fixture, lifts the t_PADIC
  factor coefficients to integers in `[0, p^k)`, and compares against
  Lean's `multifactorLiftQuadratic` output as a sorted multiset. Skips
  with exit 0 when cypari2 is not installed (mirrors the
  `if_available` mode used by `poly_flint.py`).
- Committed `conformance-fixtures/HexHensel/hensel.jsonl` so CI can
  cross-check Lean's emission against the regression sentinel before
  invoking PARI.
- Added `lean_exe hexhensel_emit_fixtures` to `lakefile.lean`.
- Added a new `oracle-pari` job to `.github/workflows/conformance.yml`
  that installs `libpari-dev` + `pari-gp` via apt, `cypari2` via pip,
  builds the emitter, diffs the fixture, and runs the oracle.
