# HexGfqRing Phase 6 API review (#3579)

## Accomplished

Polished the HexGfqRing quotient-ring API surface:

- Added `@[simp]` annotations to the inner-`reduceMod`-peeling lemmas in
  `HexGfqRing/Basic.lean`: `reduceMod_idem`, `reduceMod_self`,
  `reduceMod_add_left_reduceMod`, `reduceMod_add_right_reduceMod`,
  `reduceMod_mul_left_reduceMod`, `reduceMod_mul_right_reduceMod`,
  `ofPoly_reduceMod`. These let `simp` collapse the canonical-representative
  normalization automatically — downstream callers (e.g. HexGfqField) no
  longer need explicit `reduceMod_*_reduceMod` rewrites.
- Added docstrings to the previously-undocumented public theorems in
  `HexGfqRing/Basic.lean` (`degree_C`, `reduceMod_eq_self_of_degree_lt`,
  `reduceMod_zero`, `reduceMod_one`, `repr_ofPoly`, `ext`,
  `eq_zero_iff_repr_eq_zero`, `ne_zero_iff_repr_ne_zero`,
  `degree_repr_lt_degree`, `reduceMod_idem`, `ofPoly_reduceMod`) and the
  characterizing simp lemmas / public theorems in
  `HexGfqRing/Operations.lean` (`repr_zero`, `repr_const`, `zero_ne_one`,
  `natCast_eq_const`, `repr_natCast`, the `natCast_eq_*` family,
  `repr_add`/`repr_mul`/`repr_neg`/`repr_sub`/`repr_pow`,
  `reduceMod_add_reduceMod`, `reduceMod_mul_reduceMod`, `reduceMod_repr`,
  `ofPoly_zero_eq_zero`, `ofPoly_one_eq_one`, `ofPoly_const_eq_const`).
- Simplified the proof of `reduceMod_repr` to drop the now-redundant
  explicit `reduceMod_idem` from the `simp` set.

`lake build HexGfqRing.Basic HexGfqRing.Operations` is clean; full
project build is clean (only the pre-existing `HexPolyFp.SquareFree`
sorry and `HexGfq.CrossCheck` sorries remain). `python3 scripts/check_dag.py`
and `python3 scripts/status.py HexGfqRing` both pass. `rg` reports no
