# 2026-05-04T08:22:43Z — feature session (17cd4bad)

## Accomplished

Closed issue #2022 by adding `scripts/run-oracles-local.sh`, a local-
developer wrapper that bootstraps system + python deps for the oracle
drivers under `scripts/oracle/` and dispatches to one of them.

* `scripts/run-oracles-local.sh` — new wrapper. Detects the platform
  (NixOS / macOS / Ubuntu / unknown), maps the requested oracle to the
  required python packages and system libraries, installs only what is
  missing, creates `.venv-oracles/` once if needed, then exec's the
  driver with stdin and any extra args forwarded.
  * NixOS path re-execs the script under `nix shell nixpkgs#pari
    nixpkgs#fplll --command ...` so pip can compile the C extensions
    against the right libraries; an env-var sentinel prevents an
    infinite re-exec loop.
  * macOS path uses `brew list --formula` to skip already-installed
    formulas before `brew install`.
  * Ubuntu path uses `dpkg-query` for the same idempotency check, then
    a single `sudo apt-get install` for everything missing.
  * Per-oracle dependency map: `poly_flint` / `berlekamp_flint` /
    `bz_flint` / `matrix_flint` need only `python-flint`;
    `hensel_pari` needs `cypari2` plus libpari. An unknown / future
    driver falls back to installing the full `python-flint cypari2
    fpylll` toolchain.
* `.gitignore` — added `.venv-oracles/` so the bootstrap directory
  isn't committed.

## Verification

On carica (macOS arm64, brew):

* `bash scripts/run-oracles-local.sh poly_flint < conformance-fixtures/HexPoly/poly.jsonl`
  → `checked 6 case(s), 0 failure(s)` (first run created the venv and
  pip-installed `python-flint`).
* Re-running the same command → only the oracle output prints; no
  install steps re-execute (idempotent).
* `bash scripts/run-oracles-local.sh poly_flint --check` → flag passes
  through to the driver, same 6 cases pass.
* `bash scripts/run-oracles-local.sh hensel_pari --check` → installs
  `cypari2` (PARI was already brew-installed), 3 cases pass.
* `bash scripts/run-oracles-local.sh berlekamp_flint --check` → 36
  cases pass.
* `bash scripts/run-oracles-local.sh bz_flint --check` → 12 cases pass.
* `bash scripts/run-oracles-local.sh matrix_flint --check` → 45 cases
  pass.
* `bash scripts/run-oracles-local.sh nonexistent_oracle` → exits 1
  with a clear error pointing at `scripts/oracle/*.py`.
* `bash scripts/run-oracles-local.sh --help` and bare invocation print
  the file's leading comment block as usage.
* `shellcheck scripts/run-oracles-local.sh` → clean.

NixOS (chonk) and Ubuntu paths are unverified locally — they're behind
`uname` / `/etc/os-release` branches and weren't reachable from carica.
