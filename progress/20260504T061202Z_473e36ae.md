## Accomplished

- Added a private proof-only theorem
  `ofCoeffs_subtractScaledShift_eq_sub_monomial_mul` to
  `HexPoly/Euclid.lean`. Under the existing in-bounds hypothesis
  `hbound : ∀ j, j < q.size → shift + j < rem.size`, it states
  `ofCoeffs (subtractScaledShift rem q shift coeff) =
    ofCoeffs rem - monomial shift coeff * ofCoeffs q`,
  giving the polynomial-level reading of one in-place remainder
  update step from array long division.
- Proof strategy:
  1. `ext_coeff` reduces to a coefficient-level equation.
  2. `coeff_ofCoeffs`, `coeff_sub`, `coeff_mul` flatten the RHS to
     `rem.getD n 0 - mulCoeffSum (monomial shift coeff) (ofCoeffs q) n`.
  3. `mulCoeffSum_eq_diagonal` and `diagonalSum_eq_degree_bound`
     truncate the diagonal sum to `List.range (n + 1)`.
  4. `subtractScaledShift_getD` evaluates the LHS on the array side.
  5. A helper `hterm` shows each diagonal term equals
     `if i = shift ∧ shift ≤ n then coeff * q.getD (n - shift) 0
      else 0`, using `coeff_monomial` and
     `Lean.Grind.Semiring.zero_mul` for the off-diagonal cancellation.
  6. Fold-level induction lifts that rewrite to `(List.range (n + 1)).foldl`.
  7. Case split on `shift ≤ n`:
     - `shift ≤ n`: the conjunction collapses to `i = shift`, and
       `fold_single_index` extracts the surviving term. A further
       sub-split on `n - shift < q.size` handles the out-of-bounds
       case, where `q.getD (n - shift) 0 = 0` matches the array path
       leaving `rem.getD n 0` unchanged.
     - `shift > n`: every conditional branch is false, so the fold
       vanishes; the array path also leaves `rem.getD n 0`.
