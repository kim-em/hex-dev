# 2026-05-04 06:10Z — feature session, issue #1999

## Accomplished

- Added `HexGramSchmidt/EmitFixtures.lean`: a `lake exe
  hexgramschmidt_emit_fixtures` target emitting one `lattice` fixture
  per case plus two `result` records (`gram_det_vec`, `scaled_coeffs`)
  for seven integer bases at dimensions 4, 6, 8 (one diagonal + one
  generic basis per dimension, plus a 4×4 identity).
- Added `scripts/oracle/gs_flint.py`: python-flint driver that
  rebuilds the Gram matrix `B B^T` in `fmpq_mat`, takes leading
  principal `det()`s for `gram_det_vec`, and re-runs Lean's
  `scaledCoeffMatrix` determinant formula for each lower-triangular
  scaled-coefficient entry.  Mirrors `poly_flint.py` /
  `berlekamp_flint.py`: stdin/`--check`/path inputs, `--profile`,
  `--seed`, `--failure-dir`, `OracleMismatch` on disagreement,
  `SKIP` exit when `flint` is missing.
- Extended `Hex.Conformance.Emit` with `intListValue` and
  `intMatrixValue` so the new emit driver does not have to re-roll
  the JSON helpers privately.
- Wired the target into `lakefile.lean` and added the build/diff/run
  steps to the `oracle-pyflint` job in
  `.github/workflows/conformance.yml`.
- Committed sample at `conformance-fixtures/HexGramSchmidt/gram_schmidt.jsonl`
  (21 lines).  The fresh Lean emission diffs cleanly against it; the
  CI step replicates the regression-sentinel pattern that HexPoly and
  HexBerlekamp already use.
- Local verification: `python3 scripts/oracle/gs_flint.py --check`
  reports `checked 14 case(s), 0 failure(s)`; piping
  `lake exe hexgramschmidt_emit_fixtures` straight into the script
  also reports 0 failures.  Sanity-checked the oracle's failure path
  by tampering the `dim4/typical` `gram_det_vec` (replacing 59 with
  99); the oracle reports a mismatch, writes a JSON failure record,
  and exits non-zero.
- Step 3 of the issue (the optional `fpylll.GSO.Mat` size-reduction
  comparison) is intentionally **not** included.  The `fmpq_mat`
  cross-check already covers Gram determinants and scaled
  coefficients exactly; size-reduction parity is a separate
  HexLLL-flavoured check that belongs with HexLLL's own oracle (the
  scaled-coefficient values do not change when `sizeReduce` rewrites
  rows in place).  A follow-up issue should pick this up if/when
  HexLLL's fpylll oracle lands.
