## Current frontier

The abstract CLD column-divisibility is proven. What remains to conclude
`p^(k*d) ∣ resultant input H`:

1. **Determinant-preserving transform (bullets 1, 4).** Assemble the `d`
   shifts `t < d` of the direction `(q*X^t, -q'*X^t)` into a transform `T` and
   show `det (sylvester f g · T) = det (sylvester f g)`. Worked out:
   - Pivot for shift `t` is the **left (H-)block** column at index `d + t`,
     where `q*X^t` contributes its monic leading `1`.
   - `T` is **block-lower-triangular**: right-block (input) columns stay
     identity, so `T = [[T_LL, 0],[T_RL, I]]` and `det T = det T_LL`; `T_LL`
     is upper-triangular with unit diagonal, so `det T = 1`.
     Use `Matrix.det_fromBlocks_zero₁₂` after a `Fin.addCases` reindex (mirror
     the existing `colReduceTransform` lemmas).
   - Then feed to `pow_dvd_resultant_of_sylvester_cols`.
2. **Domain constraint (NOT in the issue body).** This syzygy construction
   needs `deg input ≥ 2*d`: the monic leading `1` of `q*X^t` sits at degree
   `d+t`, and the `d` shifts must all stay below `deg input = m` for the
   directions to be valid Sylvester columns (`q*X^t ∈ degreeLT m`). This is
   intrinsic — swapping to `res(g,f)` or pivoting in the other block does not
   remove it, because the monic factor always contributes its degree-`d`
   leading term. The `deg input < 2d` case (a "large" selected factor) needs a
   different argument (likely the per-coefficient cut/correction bookkeeping
   from `auxCutBridge`, which avoids a polynomial syzygy entirely).
3. **ZPoly → Polynomial ℤ bridge.** Derive the `Polynomial ℤ` hypotheses
   (`q` monic, `g*q - f*q' = C(p^k)*z`) from the executable substrate. The
   syzygy follows from `cldQuotientMod_congr_mul_derivative` /
   `BHKS.auxCutBridge` (which give `g*Q ≡ f*q' mod p^k` with `g`/`Q` the lifted
   factor / quotient), combined with monic-cancellation
   (`q*H ≡ f*q' mod p^k` ⟹ `H ≡ a*q' mod p^k` where `a = input/q`), plus the
   `toPolynomial` map used throughout `CLDColumnBound.lean`.
