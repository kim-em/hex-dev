# HexGramSchmidt Phase 7 reference chapter (#8198)

## Accomplished

Authored the `HexManual` reference chapter for `HexGramSchmidt` and
bumped the library to Phase 7.

- **New chapter** `HexManual/Chapters/HexGramSchmidt.lean`: a Verso
  chapter modelled on `HexManual/Chapters/HexPoly.lean`. Covers the
  dense `Hex.Matrix`-based surface and the two parallel
  `Hex.GramSchmidt.Rat` / `Hex.GramSchmidt.Int` namespaces; the core
  operations (`basis`/`coeffs` noncomputable, `gramDet`/`scaledCoeffs`
  computable) with the computable-vs-noncomputable split stated; a
  worked `#guard` example over the 3×3 integer matrix with rows
  `(1,1,0),(1,0,1),(0,1,1)` exercising `gramDet`, `scaledCoeffs`, and a
  `sizeReduce` row operation (all values copied from passing
  `HexGramSchmidt/Conformance.lean` guards); the key correctness
  theorems (`basis_orthogonal`, `basis_decomposition`, `coeffs_diag`/
  `coeffs_upper`/`coeffs_lower_projection`) in both namespaces; the
  row-operation update surface (`sizeReduce`, `adjacentSwap`,
  `basis_sizeReduce`, `basis_adjacentSwap_prev`/`_curr`); and a
  Cross-references section to the `HexMatrix` dependency, the
  `HexGramSchmidtMathlib` bridge, and `HexLLL`.
- **Wired into the aggregator**: added the import and `{include 0 ...}`
  to `HexManual.lean`.
- **Phase 7 bump**: `libraries.yml[HexGramSchmidt].done_through: 6 → 7`.

## Decisions

- The `Hex.GramSchmidt.Int.Data` structure has undocumented fields
  (`d`, `ν`), so embedding it via `{docstring}` would error. Referenced
  it with `{name}` and prose instead.
- Cross-references to `HexMatrix`, `HexGramSchmidtMathlib`, and `HexLLL`
  use plain prose/`{name}` rather than `{ref}` since no chapters exist
  for them yet (a `{ref}` to a missing tag fails to resolve). They
  resolve later when those chapters land.
