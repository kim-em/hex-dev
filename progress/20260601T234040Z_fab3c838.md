# Issue #6150: xgcd gcd bridge

## Accomplished

- Added public theorem `DensePoly.xgcd_gcd_eq_gcd` in `HexPoly/Euclid.lean`, giving a named bridge from `(xgcd p q).gcd` to public `gcd p q`.
- Rewired the two planned consumers in `HexBerlekamp/RabinSoundness.lean` and `HexPolyMathlib/Euclid.lean` to use the bridge instead of unfolding `DensePoly.gcd`.
- Verified that a global `@[simp]` annotation is not viable for this bridge: the `gcd -> xgcd.gcd` direction rewrites broad SquareFree goals into raw `xgcd` forms, while the reverse direction loops in `HexPolyFp.Quotient` because `gcd` is reducible to the same field projection.

## Current frontier

- The API bridge is available and the downstream unfold sites named in #6150 no longer use `simpa [DensePoly.gcd]`.
- The theorem is intentionally not registered as a global simp rule because both possible simp orientations break existing builds.

## Next step

- Land the PR for #6150. If a simp-normalization rule is still desired, it needs a separate design pass that avoids reducible-definition loops and broad SquareFree rewrites.
