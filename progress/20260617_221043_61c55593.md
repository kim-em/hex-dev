### Premise correction (degree = 0)

`ofIndexBelowDegree_degree_lt` as planned was **false at `degree = 0`**: there
`index : Fin (p^0) = Fin 1` is inhabited, the decoder returns the zero
polynomial, and `FpPoly.degree 0 = 0`, so `0 < 0` is unprovable. Added the
necessary `0 < degree` hypothesis and threaded it through the one consumer,
`FiniteField.reducedRepFinEquiv` (which gains a `0 < FpPoly.degree f` argument).
That equivalence is itself only sound at positive degree (`ReducedRep f` is empty
but `Fin (p^0)` is not), so the hypothesis is correct, not a weakening. The only
call site, `finEquiv`, already has `hf : 0 < FpPoly.degree f` in scope. No other
file consumes either declaration.
