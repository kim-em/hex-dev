## Accomplished

Reviewed the HO-36 proposal for parallelising
`scripts/ci/check_bench_verify_budget.sh` against the current wrapper,
`ci.yml`, `SPEC/benchmarking.md`, and lean-bench's `verify` /
`runOneBatch` implementation.

Confirmed that the current lean-bench `verify` path has no hidden
batch-in-one-child mode: `verifyOne` loops over `verifyParams`, and
each parametric check calls `runOneBatch`, which spawns one `_child`
process for a single `(registration, param)` pair.

## Current frontier

The plan is viable as a short-term CI wallclock fix, but should use a
small, explicit worker-pool implementation or `xargs -P` with careful
argument handling, deterministic log replay, temp cleanup, fail-fast
status collection, and a SPEC update because the wrapper's total would
change from summed per-library time to parallel-block wallclock.

## Next step

Implement HO-36 with `BENCH_VERIFY_CONCURRENCY` defaulting
conservatively to 2 for the first telemetry run, plus a CI env override

## Blockers

No blocker for the CI-level change. The structural fix requires an
upstream lean-bench feature that can verify many registrations and
params in one child process, or at least batch per executable.
