# Work session: HexGfqField/Bench restructure modulus_irreducible

## Accomplished

- Claimed #2258 (label: `feature`).
- In `HexGfqField/Bench.lean`, removed the universally quantified
  `private theorem modulus_irreducible (degree : Nat) :
  FpPoly.Irreducible (modulus degree) := by sorry`.  That single sorry
  was an unsound claim (the deterministic `modulus` generator is not
  designed to produce irreducible polynomials at every degree, e.g.
  `modulus 0 = #[1]`, the unit polynomial).
- Replaced it with 14 per-degree placeholder theorems
  (`modulus_2_irreducible`, `modulus_33_irreducible`, ...,
  `modulus_16385_irreducible`), one per `paramSchedule` entry plus a
  degree-2 fallback.  Each is still `sorry`, but is now a *concrete*
  claim about a specific deterministic polynomial that python-flint
  can independently verify.
- Added a private `ModulusBundle` structure (modulus + pos witness +
  irr witness) and a `bundleForN` dispatch function that maps each
  schedule `n` to the right per-degree bundle, falling back to
  `modulus 2` for any `n` outside the schedule.
- Refactored `prepOfPolyInput` and `prepBinaryInput` to call
  `bundleForN n` instead of the universal `modulus_irreducible`.  The
  derived prep functions (`prepUnaryInput`, `prepPowInput`,
  `prepZPowInput`) flow through `prepBinaryInput` unchanged.  All
  benchmark registrations (`runOfPolyReprChecksum`, `runMulChecksum`,
  `runAddChecksum`, `runNegSubChecksum`, `runPowChecksum`,
  `runInvDivChecksum`, `runZPowChecksum`, `runFrobChecksum`) continue
  to work without modification.

## Current frontier

The unsoundness scope was reduced from "universal" to "13 specific
benchmark degrees + 1 fallback".  Each per-degree sorry is a concrete,
verifiable claim about a specific polynomial in `F_8191[x]`.

## Next step

Discharge the per-degree sorries one of two ways:

1. **External verification**: run python-flint on the 14 specific
   `modulus N` polynomials and confirm/reject each irreducibility
   claim.  For any false claim, swap `modulus N` in `bundleForN` for
   a sparse known-irreducible polynomial of the same degree (e.g.
   trinomial `x^N + x + c` with `c` chosen so the polynomial is
   irreducible).
2. **Provable family**: replace `modulus N` with a family that's
   irreducible by construction (e.g. cyclotomic-style or specific
   Conway polynomials), and prove irreducibility uniformly.

The `Berlekamp.rabinTest`-to-`FpPoly.Irreducible` soundness bridge
(parent #2245's blocker) is the project-wide path forward; once that
lands, `rabinTest`-checkable polynomials can discharge each per-degree
sorry mechanically.
