## Current frontier

- The main design question is threat model honesty: `bubble` materially reduces
  accidental damage, but it does not protect a valuable Claude OAuth credential
  or host-accessible GitHub capabilities against a maliciously prompted agent.
- The cleanest near-term simplifications appear to be:
  1. reuse `bubble`'s existing transcript backing path instead of inventing a
     custom mount unless `pod` truly needs a repo-local path, and
  2. test existing `--command`/shell behavior before designing a new
     `--pod-mode`.
