**Accomplished**

- Added the AFP Berlekamp-Zassenhaus Isabelle comparator setup:
  `scripts/oracle/setup_bz_isabelle.sh` downloads and verifies the pinned AFP
  archive, builds `Berlekamp_Zassenhaus`, exports
  `Factorization_External_Interface.factor_int_poly` through the wrapper
  session, and compiles a persistent Haskell JSON-line driver.
- Added `Hex.BerlekampZassenhausBench.runIsabelleFactorChecksum` plus the
  paired Lean fixed target and a trivial-input baseline target. The Lean-side
  driver caches the comparator subprocess and canonicalises factor multisets
  before hashing.
- Added a one-time Lean-side cross-check guard over `(x-1)(x-2)`, `Phi_5`,
  and `(x^2-2)(x^2-3)` before timed Isabelle comparator calls.
- Checked for the requested `/reflect` hook; no `reflect` command or local
  command file was available in this non-interactive environment.

**Current frontier**

- The comparator is wired as a fixed benchmark on `(x^2 - 2)(x^2 - 3)`.
  Exploratory use of `smokeInput 4` exposed a live Lean-vs-Isabelle checksum
  mismatch, so that input was not used as the fixed comparator domain in this
  PR.
- With `HEX_BZ_ISABELLE` pointed at the compiled driver, the Isabelle factor
  target ran in ~9 ms on this host and the trivial baseline in ~6 ms.
