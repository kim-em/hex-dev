# HO-5d-1b-cleanup-E: route (b) on HBZ/Basic.lean

## Accomplished

Closed issue #5869 via SPEC §8 route (b) (proved unreachability):

- Added `Hex.choosePrimeData?_isSome` existence theorem with `sorry`
  body. SPEC §8 explicitly permits `sorry` here; the rigorous proof
  would need an unbounded prime search plus a Bertrand-style density
  argument against the Mignotte square-free guarantees.
- Redefined `Hex.choosePrimeData` as
  `(choosePrimeData? f).get (choosePrimeData?_isSome f)`. The total
  form now carries the §8 `unreachable-by-pipeline-invariant`
  classification.
- Removed `fallbackPrimeChoiceData`,
  `choosePrimeDataWithFallback`, and
  `choosePrimeData_eq_choosePrimeDataWithFallback`. All call sites
  switched from `choosePrimeDataWithFallback` to `choosePrimeData`
  via a single mechanical rename.
- Updated `choosePrimeData_prime` to dispatch on `choosePrimeData?_isSome`
  instead of casing into the deleted fallback.
- Updated `choosePrimeData_eq_of_choosePrimeData?_some` to a
  motive-safe proof (quantifies over the `Option.get` proof argument
  before substituting the `some` witness, so the dependent term is
  abstracted cleanly).
- Removed `factorFastWithBound_product_of_fallback_core_success_branch`
  and migrated `factorFastWithBound_product_of_some` to dispatch on
  `choosePrimeData?` cases via the existence theorem. The `none`
  arms collapse via `simp at hSome`.

`lake build HexBerlekampZassenhaus` passes cleanly: one new `sorry`
(the existence theorem) plus the pre-existing `sorry` at line 7511.
`lake build HexBerlekampZassenhausMathlib.Basic` also passes cleanly.

so the IntReductionMod arity drift is a separate (post-#5868) issue.
