## Key patterns / gotchas hit

- `GF2Poly` has no `DecidableEq`, so whole-polynomial `#guard a = b`
  fails to elaborate (`Prop` vs `Bool`). Compare via `.toWords` /
  `.degree` / `.coeff` / `.isZero` instead — same approach `HexGF2/
  Smoke.lean` already uses (`.val`, `.toWords`).
- `pureClmul` compiles to the `@[extern]` `clmul`, so `#guard`s over it
  fail under standalone `lake env lean` ("no native implementation") but
  pass inside the precompiled `HexManual` build (same as HexConway's
  `ZMod64` guards). Kept the *mandatory* executable example on the
  genuinely-pure surface (long-division gcd, shifts, XOR) validated via
  `lake env lean`; the extern-backed AES field block is pinned-correct
  by `Smoke.lean` and evaluates green in the manual build.
