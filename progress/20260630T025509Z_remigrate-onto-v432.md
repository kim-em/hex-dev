# Re-migrate matrix encapsulation onto current `main` (v4.32)

## Accomplished

The opaque-`Matrix`-structure migration (PR #8442) was complete and green
on the old baseline `0a36a98f` (branch `matrix-encapsulation`, kept as the
**reference** branch). Then `main` advanced 15 commits and diverged
structurally, so the work is being **re-migrated** onto current `main`
(`ae5870d1`, toolchain v4.32.0-rc1) on branch `matrix-encapsulation-v432`.

Why a re-migration, not a rebase: `main` never encapsulated (2114 `M[i][j]`
remain), but it (a) renamed all entry lemmas `<op>_getElem` → `getElem_<op>`,
(b) split `HexRowReduce/RowEchelon.lean` into `RowEchelon/{Contracts,Elementary}`
+ `Span.lean`, (c) independently made the row ops linear via `Vector.modify`
(commit 668daaa5), (d) uses `Matrix.identity` where the reference used `1`,
(e) bumped the toolchain. A `git rebase` produced 44 conflict files across
two divergent conventions, so the chosen path is: **take main's files as the
source of truth and apply only the encapsulation transform.**

Green on v4.32 (committed):

- `HexRowReduce` (whole lib, incl. the new split files).
- `HexDeterminant`: 12 of 13 files (all except `Plucker.lean`, WIP at ~28

## The proven mechanical recipe (per file)

1. Start from **main's** version (correct names/imports/linearity).
2. Run `scripts ... pairconv.py <file>` (in scratchpad) — converts
   `M[a][b]` → `M[(a, b)]`. Clears most files outright.
3. Fix the residue:
   - single-row `M[i]` (i : Fin n) → `getRow M i`; compound
     `(op …)[i]` → `(op …).getRow i`. (Design: **no row-`GetElem`**;
     rows go through `getRow`/`.getRow` only.)
   - `M.set` → `setRow`; `M.map` (rows) → handled in `Elementary`;
     `M[i]` row args → `getRow M i`.
   - main's `setRow_get_self`/`setRow_row_ne` (row form) →
     `getRow_setRow_self`/`getRow_setRow_ne` (in `HexMatrix.Basic`).
   - `det`/Leibniz base cases: add `getElem_rows, getElem_pair_nat,
     Fin.getElem_fin` to the `simp` so `grind` can bridge `M[(0,0)]` and
     `M.rows[0][0]`.
   - `castSucc`/`castAdd`-vs-`⟨i,_⟩` Fin mismatches: add
     `getElem_rows, Fin.getElem_fin` to the `simp`.
   - `Fin D.rank` row index needs a cast to `Fin n`
     (`⟨i.val, Nat.lt_of_lt_of_le i.isLt rank_le_n⟩`).
   - `transpose` is now `ofFn fun i j => M[(j,i)]` (NOT `col`-based), so
     proofs that `unfold Matrix.transpose Matrix.col` must drop `col` /
     use `getElem_transpose`; `getRow (transpose ..)` proofs use
     `getElem_transpose`.
   - extracting an entry from a `getRow … = 0` row fact:
     `rw [(getRow_getElem M i j).symm, hzero]; simp`.

Do **not** port reference files wholesale: reference uses divergent names
(`<op>_getElem`, `Hex.Vector.dotProduct`, `_entry` suffix, `1`/`one`,
stale imports like `HexDeterminant.Selection`→ now `Gram`). Reference is a
**solution oracle** for hard proof bodies only.
