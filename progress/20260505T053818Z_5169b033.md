## Accomplished

Discharged issue #2283 (HexLLL: drop hand-rolled `repeatXChecksum`
inner loops in `Bench.lean`). The cleanup matches the SPEC update at
PR #2281 which forbids hand-rolled inner loops in favour of
lean-bench's auto-tuner.

Edits to `HexLLL/Bench.lean`:

- Dropped the five `*HotRepeats` constants (`stateStepHotRepeats`,
  `sizeReduceHotRepeats`, `gramSchmidtCoeffHotRepeats`,
  `potentialHotRepeats`, `firstShortVectorIdentityHotRepeats`) and
  the two helper functions (`repeatIntChecksum`,
  `repeatNatChecksum`).
- Rewrote each `run*Checksum` to be a single-shot computation
  (`runSizeReduceColumnChecksum`, `runSizeReduceChecksum`,
  `runSwapStepChecksum`, `runGramSchmidtCoeffChecksum`,
  `runPotential`, `runFirstShortVectorIdentityChecksum`).
- Updated module docstring to drop the "wrapped in a fixed hot loop
  for signal-floor stability" qualifiers.
- Updated each `setup_benchmark` derivation comment to drop the
  obsolete "the fixed hot-loop repeat count is independent of `n`"
  trailers.
- Removed the now-stale `targetInnerNanos := 200000000` (200ms)
  override from five of six registrations. The override was added
  in commit 1f46ea1 alongside the hand-rolled loops as part of the
  same workaround; with the loops gone, the auto-tuner's default
  (500ms) gives it enough headroom to drive total wallclock past the
  per-spawn floor on these benchmarks.
- Bumped `targetInnerNanos := 1_000_000_000` (1s) for `runPotential`
  only, since its `prepStateInput 192+` builds a 195×391 LLLState,
  which inflates the per-spawn floor (~44ms) above the default
  500ms target's effective ceiling. The 1s budget gives auto-tuner
  inner_repeats room (≈100×) to push total wallclock above the
  10×-floor verdict threshold without changing the declared cubic
  complexity model.
