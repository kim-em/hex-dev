# 2026-05-15 22:07 UTC - HO-43 Round 2 sizeReduce relocation (d0a6d43b)

## Accomplished

Refactored `LLLState.sizeReduceColumn` in `HexLLL/Basic.lean` so its
`ν_eq` and `d_eq` proof-field discharges no longer reference any
`scaledCoeffs_sizeReduce_*` or `gramDet_sizeReduce` name. The five call
sites now use the new earlier-row-add packagings
`GramSchmidt.Int.{gramDet_rowAdd_earlier, scaledCoeffs_rowAdd_pivot,
scaledCoeffs_rowAdd_lower, scaledCoeffs_rowAdd_other_row,
scaledCoeffs_rowAdd_above_pivot}` at `c := -r`, with the
`+ (-r) * x = - r * x` and `+ -y = - y` arithmetic discharged by
`Int.neg_mul` and `Lean.Grind.Ring.sub_eq_add_neg`. `rg -n
'scaledCoeffs_sizeReduce_|gramDet_sizeReduce' HexLLL/Basic.lean` returns
zero hits.

Added three new public theorems to `HexGramSchmidt/Int.lean` mirroring
`scaledCoeffs_rowAdd_pivot` for the remaining below-diagonal cases:
`scaledCoeffs_rowAdd_lower` (column strictly left of the source column),
`scaledCoeffs_rowAdd_other_row` (rows other than the destination), and
`scaledCoeffs_rowAdd_above_pivot` (column strictly between the source
and the destination). The proofs follow the same Cramer/Bareiss pattern
as the existing pivot version, using `scaledCoeffs_eq`,
`gramDet_rowAdd_earlier`, and the corresponding `coeffs_rowAdd_*`
identities. Two new private helpers (`intCast_rat_injective_for_rowAdd`,
`scaledCoeffs_eq_fin_of_lt`) keep these proofs self-contained.

Relocated the five Round 2 size-reduce theorems
(`gramDet_sizeReduce`, `scaledCoeffs_sizeReduce_pivot`,
`scaledCoeffs_sizeReduce_lower`, `scaledCoeffs_sizeReduce_other_row`,
`scaledCoeffs_sizeReduce_above_pivot`) from `HexGramSchmidt/Update.lean`
into `HexGramSchmidtMathlib/Update.lean` (the bridge module already
holding the adjacent-swap relocations from PR #4154). Their bodies are
now three-line wrappers around the new `_rowAdd_*` packagings:
`rw [sizeReduce]` to unfold `sizeReduce b j k r = Matrix.rowAdd b j k
(-r)`, then `rw [scaledCoeffs_rowAdd_*]` (or `exact ...`), then the
`Int.neg_mul / sub_eq_add_neg` adjustment when present. The private
`scaledCoeffs_eq_fin` and `intCast_rat_injective` helpers in
`HexGramSchmidt/Update.lean` had only the relocated theorems as
callers, so they were deleted along with the relocated block. The
bridge already has its own `intCast_rat_injective_local` for the
adjacent-swap theorems; the new size-reduce wrappers do not need any
injectivity helper because their proofs stay integer-valued.

Refreshed the bridge module's top-of-file docstring to drop the
"this module currently holds only the `adjacentSwap`-side relocations"
note and describe the present scope (both size-reduce and
adjacent-swap relocations).
