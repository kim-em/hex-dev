# Progress: issue #8677 — closed-form `defaultFactorCoeffBound`

## Accomplished

`ZPoly.defaultFactorCoeffBound` (`HexPolyZ/Mignotte.lean`) recomputed the
loop-invariant bignum norm `coeffL2NormBound f` inside every one of its
`O(deg^2)` `mignotteCoeffBound f k j = binom k j * coeffL2NormBound f` terms.
Since `binom k j` peaks at the central binomial, the whole double max collapses
to `binom n (n / 2) * coeffL2NormBound f` (`n = degree f`) — one norm, one
binomial.

Landed as the SPEC design-principle-11 twin: the public spec is now
`noncomputable` (kernel `decide` still reduces it), and the compiled runtime uses
`defaultFactorCoeffBoundImpl` (the closed form) via a proved `@[csimp]` swap
`defaultFactorCoeffBound_eq_impl`. No downstream proof re-keying: every existing
`defaultFactorCoeffBound_*` lemma reasons about the unchanged spec.

Supporting Mathlib-free monotonicity (`binom_le_central : j ≤ k → k ≤ n →
binom k j ≤ binom n (n / 2)`) is proved by bridging `binom` to the Mathlib-free
Pascal `Hex.Nat.choose` (new import `HexArith.Nat.Prime`, acyclic) and porting
central-binomial unimodality/monotonicity. `binom`'s own definitional symmetry
avoids needing a choose-symmetry lemma.

Verification: whole graph green (`HexPolyZ`, `HexBerlekampZassenhaus`,
`HexPolyZMathlib`, `HexBerlekampZassenhausMathlib`, `hexbz_bench`,
`hexbz_emit_fixtures`). A standalone check confirms the compiled csimp path is
**value-identical** to the original double loop over a degree grid
(0,1,2,3,5,8,12,16,20,24,25 × shifts) — so conformance fixtures / certification
are untouched — and ~319× faster on the deg-24 split family (~16 ms → ~50 us per
call, matching the issue's "few hundred times cheaper").
