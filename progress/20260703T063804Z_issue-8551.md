## The mistake, and the doc fix

First attempt started a **full 9-system sweep** (rebuilding both Isabelle AFP
heaps) to get an all-systems cross-check record. That was wrong: the plotter
merges newest-per-system, so adding one comparator only needs `--systems pari`.
Kim caught it. Rewrote the "Reproducing" guidance in **both** the report and
`HexBerlekampZassenhaus/SPEC/hex-berlekamp-zassenhaus.md` so the principle is
unmissable: adding or refreshing one system is a single-system run; re-running
the expensive external comparators (especially Isabelle) is never needed to add
one; full-board runs are only for a host's first record or a corpus change.
