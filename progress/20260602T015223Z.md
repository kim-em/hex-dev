# Guard against inverted issue-graph dependencies

## Accomplished
Specified a guard (PLAN-only; Phase 0 builds the tooling) so an issue
about an upstream library can never sit permanently `blocked` on a
downstream-library issue — the pathology behind #2637.

- `PLAN/Conventions.md`: added a machine-readable `library:` line to
  the standard issue body (PascalCase libraries.yml key), and a new
  §"Inverted dependencies are void". An inverted `depends-on:` edge
  (downstream-library dep of an upstream issue) is provably not a real
  blocker by the import DAG. The guard keeps such edges out of issue
  bodies: reject at creation, scrub from the backlog (remove the line,
  clear stale `blocked`, route to `replan`). No SPEC edits by agents,
  no halting for a human — the pipeline keeps rolling.
- `PLAN/Phase0.md`: specified the reusable offline predicate
  `may_import(l_a, l_b, libraries)` in scripts/libgraph.py, shared by
  check_dag.py (file imports) and the orchestration (issue edges). No
  new CI job, no per-push GitHub API calls.
