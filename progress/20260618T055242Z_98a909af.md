### Premise correction (#7869) and executable helpers

The directive's arithmetic needed the per-column bound `2·|coord_j| ≤ |S|`,
but #7869 (`two_mul_natAbs_sum_psiCut_period_le`) was *stated* with the weaker
`≤ |S| + 1`. The `+1` is fatal at full support `|S| = r` (forces
`4·‖v‖² ≤ 4r + n·(r+1)²`, exceeding `bhksCutRadiusSq4 = 4r + n·r²`). The
lemma's own proof already established the strict `< |S| + 1`, i.e. `≤ |S|`, so
I tightened the stated conclusion to `≤ |S|` (one-line change; the raw sibling
`two_mul_natAbs_sum_psiCut_le` already concludes `≤ |S|`). This is "fix the
API", not a weakening — #7874 was the first consumer.
