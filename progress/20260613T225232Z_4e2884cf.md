## Directive/premise check (the finding)

The genuine deliverable is `resultant_divisible_by_p_pow` for the concrete
`BHKS.auxiliaryPolynomialWithCorrections`. There is no sound route:

- The abstract column theorem `pow_dvd_resultant_of_sylvester_cols`
  (`Resultant.lean:184`) consumes raw Sylvester-column data — a concrete matrix
  `A`, injective `cols`, `hdet : A.det = sylvester.det`, `hcols : p^k ∣ A i (cols j)`.
  Nothing derives that data from a CLD polynomial.
- #6947/#6952 landed only *coefficient-level* CLD facts
  (`cldCoeffs_getD_of_lt` `Basic.lean:4750`, `cldQuotientMod_coeff_decomp_of_lt`
  `Basic.lean:4771`, the Lemma 5.1 bound `abs_cldCoeffs_le_bhksCoeffBound`
  `Basic.lean:5041`). These feed the auxiliary l2norm/Mignotte estimate, not any
  Sylvester column or resultant divisibility. `BadVector.lean` has no
  `sylvester`/`cols`/`colReduce` reference except the Hadamard bound (`:80`).
- The only resultant-level theorems that *conclude* divisibility
  (`commonFactor_dvd_resultant` `:851`, `..._of_monic_cofactor` `:923`,
  `pow_dvd_resultant_of_map_dvd` `:992`) all require `q ∣ aux mod p^k` — the
  false premise the issue itself forbids. The CLD construction exists precisely
  because `q ∤ aux mod p^k`; the `colReduceTransform` family (`:685`, `:797`) is
  the shared-modular-factor reduction and does not apply.

So the missing piece is the BHKS Lemma 3.2 valuation core: prove the CLD aux
polynomial yields `d = localFactorDegree` Sylvester columns divisible by `p^k`
under a determinant-preserving transform driven by the input's selected-factor
cofactor `a` (not an aux cofactor `b`). That is new math, not wiring. The
current constructors already document the resultant field as "the intended
boundary before the full BHKS Lemma 3.2 proof" (`BadVector.lean:1160`).

Deliverable #2's fallback ("reduce the field to genuinely external hypotheses")
would only relocate an equivalent assumption to a heavier matrix-valued API and
push it up through every constructor, constructing nothing — declined as it
hides the gap rather than closing it.
