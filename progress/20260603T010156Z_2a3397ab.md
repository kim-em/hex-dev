# 2a3397ab — fused `(a*b - c*d) / e` FFI for stepScaledRows inner cell

Closes #6408. See SPEC/profiling.md and reports/hex-lll-performance.md for
context. Per-turn entry per `.claude/CLAUDE.md`.

## Accomplished

- Added `HexMatrix/ffi/fma_div_exact.c` with `lean_hex_int_fma_div_exact(a, b, c, d, e)`.
  - GMP path: five `lean_int_to_mpz` extracts, three mpz scratch slots
    (`lhs`, `rhs`, `q`), one `mpz_mul`/`mpz_mul`/`mpz_sub`/`mpz_divexact`
    chain, one heap mpz allocated for the result. Replaces the four heap
    `Int` boxes the Lean-level expression emits.
  - Non-GMP fallback: composes the unfused `lean_int_mul`/`lean_int_sub`/
    `lean_int_div_exact` so the value still matches the Lean definition
    when GMP is absent.
- Extended `lakefile.lean`: added `hexMatrixOTarget`, `extern_lib
  hexmatrixffi`, and `moreLinkArgs` on the `HexMatrix` lib (mirroring
  the existing `HexArith` pattern; same `-lgmp` link).
- Declared `Hex.Matrix.fmaDivExact (a b c d e : @& Int) : Int :=
  exactDiv (a*b - c*d) e` in `HexMatrix/Bareiss.lean` with
  `@[extern "lean_hex_int_fma_div_exact"]`. Added two equality
  lemmas: `fmaDivExact_eq_exactDiv` (rfl) and `fmaDivExact_eq_divExact`
  (under divisibility, matches `Int.divExact`).
- Rewrote the inner-cell expression in
  `HexGramSchmidt.Int.stepScaledRows` to call the fused helper.
- Patched two dependent proofs in `HexGramSchmidt/Int.lean`:
  - `stepScaledRows_row_at_trailing`: added `Matrix.fmaDivExact` to the
    `simp` set so the unfolded fold normalises back to the
    `Matrix.exactDiv` form the lemma's RHS expects.
  - `stepScaledRows_rows_size`: the local `f` definition now uses the
    same `fmaDivExact` form that `unfold stepScaledRows` produces, so
    the `rw [getElem!_foldl_modify_*]` lemmas match.
