## Current frontier

The plan's structural direction is sound, but the in-process rewrite
should avoid pretending that task timeout can kill CPU-bound Lean code.
Lean's available async cancellation is cooperative; a runaway benchmark
inside the same process can keep running after a race/timer result is
reported, or block the executable outright if run synchronously.

`LeanBenchTestVerify.testEndToEndFailure` currently expects the fake
spec path to exercise child-spawn failure; that test needs rewriting
for registry-lookup failure under in-process verify.
