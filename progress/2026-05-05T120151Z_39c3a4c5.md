## Accomplished

- Added `HexGF2/CommonIrreducibility.lean` carrying a Rabin-style
  certificate for the AES Rijndael modulus `X^8 + X^4 + X^3 + X + 1`
  (`ofUInt64Monic 0x1B 8`). The cert is built from explicit
  `xpow2kMod`/`xgcd` calls inside an array literal so kernel reduction
  bypasses `Array.map`, and a single `set_option maxRecDepth 4096 in
  decide` closes `checkIrreducibilityCertificate aesModulus aesCert =
  true` in ~2 s.
- Discharged the AES `private theorem aesIrreducible` sorry in four
  fixture files: `HexGF2/Bench.lean`, `HexGF2/Conformance.lean`,
  `HexGF2/EmitFixtures.lean` (`aesIrr` only), and `HexGF2/Smoke.lean`.
  Each now reuses `Hex.GF2Poly.aes_modulus_irreducible`.
- Added a kernel-friendlier linear cert checker variant to
  `HexGF2/Irreducibility.lean`: `checkPowChainLinear`,
  `checkIrreducibilityCertificateLinear`, plus their soundness
  theorems against `rabinTest` mirroring the existing non-linear
  proofs. `HexGF2/RabinSoundness.lean` exposes the corresponding
  `checkIrreducibilityCertificateLinear_imp_irreducible` bridge to
  `GF2Poly.Irreducible`. The linear checker recovers `O(n)` chain
  reductions versus the existing `O(n^2)` recomputation.

## Next step

Sub-issue #2360 covers the GHASH-modulus discharge. Empirically a
single deg-127 `sqMod` step exhausts ~4 M heartbeats in kernel
reduction, so a monolithic `decide` on `checkIrreducibilityCertificate
Linear gf2nPolyModulus cert` is not viable (~10 GB+ memory, 100 M+
heartbeats without converging). The proposed approach is to split the
cert verification into one-`sqMod`-per-`decide` step proofs and stitch
them together manually. The infrastructure landed here is what that
follow-up will build on.
