# Split the Mathlib bridge: HexMatrixMathlib → four bridges (PR3)

## Accomplished

Third PR of the matrix-family split: partition the Mathlib bridge layer to
mirror the computational split, so each computational library has its own
Mathlib bridge.

Bridge DAG:
```
hex-matrix-mathlib       -> hex-matrix
hex-row-reduce-mathlib   -> hex-row-reduce, hex-matrix-mathlib
hex-determinant-mathlib  -> hex-determinant, hex-bareiss, hex-matrix-mathlib
hex-bareiss-mathlib      -> hex-determinant-mathlib   (and hex-bareiss transitively)
```

- **HexMatrixMathlib** (base) keeps `Basic.lean` (`matrixEquiv` + the row-op
  correspondence lemmas). Its import was over-broad (`HexRowReduce.RowEchelon`)
  — `Basic.lean` uses no row-reduce symbol, only base `Elementary` row ops — so
  it now imports `HexMatrix`. Severing that makes the base bridge depend only on
  the computational base.
- **HexRowReduceMathlib**: `RankSpanNullspace` (rank/span/nullspace bridge).
- **HexDeterminantMathlib**: `Core`, `CorePlucker`, `CoreTransport`,
  `DesnanotJacobi` (the `det_eq` determinant correspondence). `CoreTransport`
  imports the executable `hex-bareiss` + `hex-determinant`, so this bridge
  depends on those computational libs (Codex's noted tangle, modelled
  explicitly).
- **HexBareissMathlib**: `Bareiss` (no-pivot bordered-minor invariant,
  `bareiss_eq_mathlib_det`) plus the `bareissDet_eq_det` / `bareiss_eq_det`
  headline theorems moved from the old `Determinant.lean` umbrella.

Internal `namespace HexMatrixMathlib` is kept across all four libraries, so
downstream symbol references (`HexMatrixMathlib.bareiss_eq_det`, `matrixEquiv`,
…) are unchanged; only module paths and imports moved.

**Downstream** (Codex's broader-consumer finding): `HexGramSchmidtMathlib` uses
`bareiss_eq_det` / `bareiss_eq_mathlib_det` / `BareissNoPivotInvariant`, now all
in `hex-bareiss-mathlib`; its `Int/GramDet` import switched
`HexDeterminantMathlib` → `HexBareissMathlib` (which re-exports the determinant
bridge), and its dep updated accordingly. `HexLLLMathlib/Basic` →
`HexRowReduceMathlib.RankSpanNullspace`.

lakefile gains three `lean_lib` targets; `libraries.yml` gains three mathlib
entries and updates the downstream deps; the `HexMatrixMathlib` SPEC is split
into four per-library SPEC files.

Verified green: `lake build` (4101 jobs, all four bridges + GramSchmidtMathlib +
LLLMathlib + HexBerlekampZassenhausMathlib + HexManual), and validators
`check_dag`, `check_phase4`, `check_copyright_headers`,
`conformance_targets --check`, `check_benches_mathlib_free`.
