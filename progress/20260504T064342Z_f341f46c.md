## Accomplished

Closed the gap left by issue #2023 (sibling of #1994): wired up a
python-flint factorisation oracle for `HexBerlekampZassenhaus`.

* `HexBerlekampZassenhaus/EmitFixtures.lean` — integer-polynomial
  fixtures spanning the four shapes the SPEC distinguishes
  (Mignotte-bounded irreducibles, reducible 2-3-factor products,
  content > 1, polynomials with many local factors at a good prime).
  Twelve cases in total at degrees 4, 5, 6, 10, 16.  All cases were
  chosen to keep `coeffL2NormBound` small enough that the production
  lift completes quickly under the current pipeline state.
* `scripts/oracle/bz_flint.py` — re-factors each Lean-reported
  component through `fmpz_poly.factor()` and compares the combined
  `(content, irreducible-multiset)` signature against a direct
  factorisation of the input polynomial.  This canonicalises the
  cross-check whether or not Lean fully splits the input — a content
  mismatch, missing factor, or bogus factor trip the oracle, while a
  trivially-unsplit `[f]` collapses to the same signature as direct
  factorisation (catching corruption regressions while remaining
  meaningful even before the BZ pipeline performs full irreducible
  factorisation modulo the chosen good prime).
* `conformance-fixtures/HexBerlekampZassenhaus/bz.jsonl` — committed
  sample replayable via `python3 scripts/oracle/bz_flint.py --check`.
* CI: added `hexbz_emit_fixtures` build + diff + oracle run to the
  `oracle-pyflint` job in `.github/workflows/conformance.yml`,
  matching the pattern used by `HexBerlekamp` and `HexMatrix`.

## Current frontier

The `HexBerlekampZassenhaus` library is at `done_through: 0`; full
public-API factorisation (`Hex.factor`) currently returns `[f]` for
many reducible inputs because `factorsModP` is built from
`squareFreeDecomposition` (one bucket per multiplicity, never a
true irreducible split).  The oracle is structured so it stays
useful through that incompleteness — re-factoring each Lean
component via `fmpz_poly.factor()` canonicalises the cross-check —
and is ready to detect any product or content corruption introduced
in the upcoming Phase-2 factorisation work.

## Next step

When the BZ pipeline begins producing genuinely-split factor arrays
(Phase 2: irreducible factorisation modulo `p`, then real LLL-based
recombination), the existing fixtures will start exercising the
LLL-only branch via `lll/x5_minus_x` and `lll/x7_minus_x`.  Add more
fixtures with deliberately large coefficients only after the lift
performance bottleneck (`multifactorLiftQuadratic` at large `k`) is
addressed.
