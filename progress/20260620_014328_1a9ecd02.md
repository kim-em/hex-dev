## Documented exclusion (directive-as-hypothesis)

The directive listed `p_odd_nat` (`Montgomery/Context.lean` L205) as one
of the two definite literal-equation promotions, but the real build
**rejects** it from `grind =` (hard error). Left `@[simp]`-only:

```
invalid pattern(s) for `p_odd_nat`
  [@HMod.hMod `[Nat] `[Nat] `[Nat] `[instHMod] (UInt64.toNat #1) `[2]]
the following theorem parameters cannot be instantiated:
  ctx : MontCtx p
```

Root cause: `p_odd_nat (ctx : MontCtx p) : p.toNat % 2 = 1` has a
conclusion (and thus E-matching pattern `p.toNat % 2`) that mentions only
`p`, never `ctx`. The *data*-typed parameter `ctx : MontCtx p` cannot be
instantiated from the pattern, so grind rejects it. Contrast
`mk_p_odd_nat`, whose uninstantiated parameter `hp : p % 2 = 1` is a
*proof* and is therefore tolerated.

General rule for future sweeps: a lemma whose conclusion drops a
**data** (non-proof) parameter is rejected by `grind =`; dropping a
**proof** parameter is fine.
