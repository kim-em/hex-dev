## Accomplished

- Investigated whether `pod` sub-agents can sensibly be wrapped in `bubble`
  containers (planning conversation; not a hex code change). Findings filed
  as a plan at `~/.claude/plans/can-you-double-check-jiggly-puzzle.md`.
- Got a Codex second opinion that pruned the original plan: of the four
  bubble-side changes I'd proposed (B1–B4), only B1 is actually needed; the
  rest are already covered by existing flags or by writable mounts.
- Spun up a sandbox project at `~/projects/pod-bubble-experiment/`, pushed
  to https://github.com/kim-em/pod-bubble-experiment, with a tiny `PLAN.md`
  describing three trivial deliverables that exercise each pod↔bubble seam
  (file edits, `make`, `git`, `gh pr create`, transcript JSONL).
- Phase 0 / Phase 1 measurements on Colima:
  - Bubble base image build (one-time): 3:14.
  - Cold spawn (per-agent cost): ~25s.
  - Warm reattach: ~3s.
  - Trivial `claude -p` round-trip inside the bubble: works; auth comes
    via the mounted `~/.claude/.credentials.json`.
  - Transcript JSONL exfiltrated to host at
    `~/.bubble/ai-projects/<bubble-name>/<encoded-cwd>/<uuid>.jsonl`.
  - `gh api repos/kim-em/pod-bubble-experiment` succeeds via the bubble
    auth proxy (per-repo REST works at default security level).
  - User's full `~/.claude/CLAUDE.md`, `skills/`, `commands/`,
    `settings.json` all leak into the bubble — confirms the granular
    `--no-claude-config` request is necessary.
- Filed five bubble issues to capture friction the experiment surfaced:
  https://github.com/kim-em/bubble/issues/264 (granular config mount),
  /issues/265 (`--command` quoting),
  /issues/266 (`--ephemeral` auto-pop),
  /issues/267 (`--shell --command` cwd in `/home/user`),
  /issues/268 (`GH_TOKEN` only in login shells).
