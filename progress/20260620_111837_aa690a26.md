## Key decisions / patterns

- **No capstone equivalence.** Unlike `HexPolyMathlib`/`HexModArithMathlib`
  (which end on a bundled `≃+*`), this bridge has none — it is a family of
  proof-only correctness/uniqueness theorems about the lifting *procedure*.
  The intro states this explicitly so a reader does not hunt for an
  `equiv`.
- **Worked examples typecheck, not eval.** The surface is `noncomputable`,
  so the two examples invoke public lemmas by name
  (`coeff_map_intCastRingHom_eq_zero_iff_dvd`,
  `toPolynomial_monic_of_dense_monic`) — imitating the `HexPolyMathlib`
  `by simp` example style. Kept binders in `variable` blocks and every
  code line under the 60-column cap.
- **Cross-refs resolve.** `{ref "hex-hensel"}` (computational counterpart)
  and `{ref "hex-poly-mathlib"}` (transfer-map dependency) both exist.
