# 2026-05-04 08:01Z — feature session, issue #2013

## Accomplished

Closed #2013 by appending 6×6 fixtures to
`HexGramSchmidt/Conformance.lean`, hitting the SPEC `core`
matrix-dimension band (target 6, ceiling 8) while preserving the
existing 2×2 / 3×3 / 3×2 typical / edge / adversarial structure:

- `bigTypical` — `J − I` complement-of-identity at 6×6. Each row has 5
  ones; distinct rows share 4 columns. `B Bᵀ = 5·I + 4·(J − I)` with
  eigenvalues `4k + 1, 1, …, 1` on the leading `k×k` Gram, so the Gram
  determinant chain is `1, 5, 9, 13, 17, 21, 25` — none zero, exercising
  Bareiss inner-loop work at every prefix step.
- `bigZero` — 6×6 zero matrix; gives the empty-prefix `1` then a
  collapsed chain.
- `bigDependent` — 6×6 with row 1 = 2·row 0 and rows 2..5 the unit
  basis on cols 2..5. Mirrors the smaller `dependent` shape: leading
  Gram chain is `1, 4, 0, 0, 0, 0, 0` (singular from `k = 2` onward
  even though scaling rows 2..5 are independent).

Added `#guard`s:

- `gramDetVecAgrees` on each fixture (validates per-row-prefix
  consistency between `gramDetVec` and `gramDet`).
- Specific `gramDet` values across the chain for `bigTypical` (k = 0..6),
  plus the boundary values for `bigZero` and `bigDependent` covering the
  singular prefix transition.
- `scaledCoeffs` diagonal entries (= leading Gram chain),
  representative sub-diagonal entries away from row 0
  (`[3][1] = 4`, `[5][4] = 4`), and three above-diagonal zero checks
  on `bigTypical`.
- `sizeReduce` of the last row by row 0 with coefficient 1 — turns
  `(1,1,1,1,1,0) − (0,1,1,1,1,1)` into `(1,0,0,0,0,−1)`, validated at
  three entries.
- `adjacentSwap` of rows 4 and 5 of `bigTypical`, validated at the
  exchanged unique-zero columns.
- Adjacent-swap helper formulas at the last row of `bigTypical`:
  `denom = 21`, `pivot = 4`, `numerator = 25·17 + 16 = 441`,
  `quotient = 21`.

build with a `#guard` failure; reverted.
