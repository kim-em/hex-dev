## Key decisions / patterns

- **No `{ref "hex-gf2"}`.** The `HexGF2` chapter (#8228) had not landed at
  author time, so no `hex-gf2` tag exists and a `{ref}` to it is a Verso
  build error. Per `PLAN/Phase7.md` ("a stub link is acceptable and
  resolves later"), `HexGF2` is named in prose with a "forthcoming"
  note instead. `{ref "hex-conway"}`, `{ref "hex-gfq-field"}`,
  `{ref "hex-gfq-ring"}` all resolve (chapters exist).
- **`{name}` not `{docstring}` for the classes.** `Conway.CommittedEntry`
  and `Conway.PackedGF2Entry` have undocumented fields, and `{docstring}`
  on a class requires every field documented — so they are embedded with
  `{name}` and described in prose. `{docstring}` is used only for
  documented defs/theorems (`GFq.ofPoly`, `GFqC.ofPoly`,
  `GFqC.modulus_eq_gfq`, `GFq.frob`, `packedGF2FpPoly`, `GF2q.ofWord`,
  `GF2q.toGFq`, `packedGF2Entry_2_1_irreducible`).
- **`#guard` over field elements needs `repr`.** `GF2q` has no
  `DecidableEq`, so `#guard (a * a⁻¹ = 1)` fails to evaluate; phrase every
  check as `GF2q.repr … = <word>` (UInt64, decidable). The extern packed
  arithmetic evaluates green inside the precompiled `HexManual` build.
- **Verso bold is single `*`.** `**x**` warns; use `*x*`.
