# Fast small-mod singleton raw guarded irreducibility (#8097)

## Accomplished

Added `factorFastFactorsWithBound_raw_guardedIrreducible_of_smallModSingleton`
to `HexBerlekampZassenhausMathlib/IntReductionMod.lean`, the missing fast-path
raw guarded irreducibility producer for the small-mod singleton branch
(`primeData.factorsModP.size ≤ 1`). It completes the trio of guarded raw
producers consumed by the default factor irreducibility capstone successor
#8068, alongside `factorFastFactorsWithBound_raw_irreducible_of_constant`
(constant arm) and `factorFastFactorsWithBound_raw_guardedIrreducible_of_recoveredLift`
(BHKS core-success arm).

Conclusion is the corrected guarded shape:
`∀ raw ∈ rawFactors.toList, shouldRecordPolynomialFactor (normalizeFactorSign raw)
= true → ZPoly.Irreducible raw`.

## Decisions / patterns

- `Hex.reassemblePolynomialFactors` is `private` to `HexBerlekampZassenhaus/
  Basic.lean`, so the Mathlib layer cannot name it to state the branch value
  equation directly (my first draft did and failed with "Unknown identifier").
  Instead the proof obtains the value equation through the public theorem
  `Hex.factorFastFactorsWithBound_branch_of_choosePrimeData?_some`, whose
  conclusion already carries `some (reassemblePolynomialFactors …)` in disjuncts
  (c)/(g). The non-singleton disjuncts are eliminated by the branch markers
  (`hdeg`, `hB_pos`, `hsmall`, and the `hquadratic` dispatch guard). The private
  reassembly term is thus only ever referenced through that theorem's
  conclusion, never written.
- Using the branch theorem also sidesteps the `whnf` heartbeat timeout that an
  in-proof `unfold Hex.factorFastFactorsWithBound` triggered (the heavy unfold
  is paid once inside the already-compiled branch theorem).
- Irreducibility of each raw factor reuses the exact substrate from
  `factor_small_mod_singleton_branch_entry_irreducible`'s proof:
  `squareFreeCore_irreducible_of_smallModSingletonBranch` →
  `reassemblyExpansionComplete_singleton_of_irreducible_of_pos_lc` →
  `reassemblePolynomialFactors_factor_irreducible_of_complete_and_core_irreducible`.
  Since the singleton-core has positive degree (`hdeg`) it is genuinely
  irreducible (not the unit `1` of the constant arm), so the recorded-factor
  guard is discharged for free and is not consumed by the proof.
