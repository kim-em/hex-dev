# Work session: HexBerlekamp.DistinctDegree bucket invariants

## Accomplished

- Claimed #2234 (label: `feature`).
- In `HexBerlekamp/DistinctDegree.lean`, replaced the
  `distinctDegreeFactor_bucket_invariants` proof hole with a real
  proof, supported by a private loop-invariant lemma
  `distinctDegreeLoop_bucket_invariants`.
- Reformulated `DegreeBucket.matchesFrobeniusDegree` from
  `gcd bucket.factor (frobeniusDiffMod ...) = bucket.factor` to the
  divisibility statement `bucket.factor ∣ frobeniusDiffMod ...`.
  Rationale: the project's executable `DensePoly.gcd` is unnormalised,
  so the equality form would require a `gcd_eq_left_of_dvd` bridge that
  is provably false (e.g. `gcd X (2·X) = 2·X` over `F_3`). The
  divisibility form captures the morally-correct invariant and follows
  from `DensePoly.gcd_dvd_right`. The doc-comment on the new definition
  records the rationale.
- The loop-invariant proof is by induction on `fuel`, threading the
  joint invariant
  `currentFrob = FpPoly.frobeniusXPowMod f hmonic d ∧ 0 < d ∧
    every bucket already in the accumulator satisfies the bucket
    invariant`. The successor step uses `frobeniusXPowMod_succ`
  (landed in #2233) to advance `currentFrob` to
  `frobeniusXPowMod f hmonic (d + 1)`.
- Added `[ZMod64.PrimeModulus p]` to the public theorem signature so
  the `frobeniusXPowMod_succ` dependency is satisfiable. This matches
  the existing `prod_distinctDegreeFactor` signature.
- Verified:
  - `lake build HexBerlekamp.DistinctDegree`
  - `lake build HexBerlekamp`
  - `python3 scripts/check_dag.py` (exit 0)
  - `git diff --check` (clean)
  - No new `axiom`, `native_decide`, `sorry`, `TODO`, or `FIXME` in
    `HexBerlekamp/DistinctDegree.lean`.
