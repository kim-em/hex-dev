# issue #8691: Phase 2 (proof) and Phase 3 (wiring) complete; SD4 end-to-end win is null

## Accomplished

PR https://github.com/kim-em/hex-dev/pull/8697 (branch `issue-8691`), all green,
no `sorry`/`axiom`. Pushed through commit `63fa1a38`.

- **Phase 2 (byte-identical proof), done.** `cldQuotientModWord?_eq` in
  `HexBerlekampZassenhausMathlib/WordCld.lean` proves, for monic `g` of positive
  degree under `p^a < 2^64 ∧ odd ∧ 1 < p^a`, that `cldQuotientModWord? f g p a =
  some (cldQuotientModBignum f g p a)`. Both quotients transport to the same
  `numerator /ₘ divisor` in `ZMod (p^a)[X]` (via `map_divMod_monic` and the
  coefficient casts) and their canonical-`[0,p^a)` outputs agree coefficientwise.
  Final leaves `gW` monic / positive-degree closed by pinning
  `(toWMap ctx g).size = g.size` (the mapped last coeff `toW 1 = 1 ≠ 0`, no trim).

- **Phase 3 (wiring), done.** Split the bignum reference out as
  `cldQuotientModBignum`; `cldQuotientMod` now dispatches to the word path under
  the decidable guard `p^a < 2^64 ∧ odd ∧ 1 < p^a ∧ Monic g ∧ 0 < deg g`, falling
  back to bignum otherwise. `cldQuotientMod_eq_spec` proves the dispatch equals
  `cldQuotientModBignum` for every input; `cldQuotientMod_congr_mul_derivative`
  (CLDColumnBound) now derives its old `rfl` through it.

  **Why a redefinition, not `@[csimp]`.** The correspondence proof is
  Mathlib-dependent, so a `@[csimp]` lemma can only live in the Mathlib layer and
  would not reach the Mathlib-free lattice tier / bench (compiled earlier).
  Putting the guarded dispatch in `cldQuotientMod`'s own Mathlib-free definition
  delivers the fast path to every consumer; the sole cost was reproving one `rfl`.

## Current frontier — the honest measurement

Re-measured `factorLatticeAdvSwinnertonDyerSD4` cold-call wallclock (12 fresh
processes each), fast path vs. bignum-neutered:

- fast path ON:  ~20.5 ms (20.44–20.58 ms)
- word path OFF: ~20.5 ms (20.42–20.86 ms)

**No measurable end-to-end win at SD4.** A `dbg_trace` confirms the word path
*does* fire many times during SD4 (moduli 25, 3125, ..., up to 1.49e18 < 2^64),
so the null result is not a missed guard: the CLD quotient simply is not a
bottleneck for SD4. The lattice tier's ~20 ms is dominated by the earlier stages
(modular split, Hensel lift, LLL). The word kernel is faster in isolation
(byte-identical, ~1.8x on the isolated quotient at the 11^13 modulus), and the
dispatch adds no measurable overhead (ON == OFF), so it is a zero-risk net
positive that will help on CLD-heavy inputs — but SD4 does not showcase it.
