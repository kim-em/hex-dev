## Current frontier

The product-check upper bound is independent of the BHKS lattice coordinate,
but the lower count bound is not: it links the true-support partition count to
the emitted array only through the executable indicator array for the same
projected-row lattice.

The current direct producer stack has monic-coordinate recovered-lift machinery
and toMonic short-vector packaging, but the executable fast loop exposes the
hybrid lattice `bhksLatticeBasis core ... (toMonicLiftData core ...).liftedFactors`.
Closing the arm still needs a cut producer for that hybrid lattice, or a
separate proof relating the pure-M1 and hybrid indicator counts.

## Next step

`factorFastCoreWithBound_some_indicatorCandidates`, using the existing
`RecoveredLift` / period-adjusted short-vector route as the model. Avoid the
pure-M1 count-transfer route unless a same-lattice indicator-count comparison
is first proved.
