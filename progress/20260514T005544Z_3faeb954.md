# Progress - feature session, issue #3878

## Accomplished

Claimed #3878 ("HexMatrix: prove unscaled Mathlib-free Desnanot-Jacobi
from scaled identity"), then skipped it after confirming a placement
conflict with the project's hard rule on Mathlib-free libraries.

The scaled prerequisite from PR #3880 is on `main` and reachable:
- `Hex.Matrix.det_auxAdjugateM` (`HexMatrix/Determinant.lean:9352`)
- `Hex.Matrix.det_desnanot_jacobi_mul` (`HexMatrix/Determinant.lean:9409`)

The unscaled identity asked for by the issue needs `det M *
(corner_diff) = det M * (det M * det_interior)` to be cancelled into
`corner_diff = det M * det_interior` over an arbitrary
`Lean.Grind.CommRing R`. That cancellation in a generic commutative
ring has only two known proofs: domain reasoning, or the universal-
polynomial trick over `MvPolynomial _ Int`. The Mathlib version in
`HexMatrixMathlib/Determinant/DesnanotJacobi.lean` uses the second.

Mathlib-free `MvPoly`/`NoZeroDivisors`/`IsDomain` infrastructure does
not exist locally — only `ZMod64 p` and `PolyQuotient` carry
`Lean.Grind.CommRing` instances, and neither helps. Building it would
contradict the explicit rule in
[PLAN/Conventions.md §"Library placement is a hard precondition" Q2]:

> Mathlib-free libraries cannot host a proof whose shortest path goes
> through `Matrix.adjugate`, the universal polynomial ring,
> `MvPolynomial`, `IsIntegralClosure`, or similar.

It also contradicts [SPEC/Libraries/hex-matrix.md lines 131-133]:

> Do not reprove Desnanot-Jacobi locally - track [mathlib4 PR #37716].
> If merged, import it; otherwise prove locally using Mathlib's
> `Matrix.adjugate`.
