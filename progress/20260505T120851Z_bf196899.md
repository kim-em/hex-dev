Claimed #2358 (sub-issue of human-oversight #2245) and shipped the

`HexBerlekamp/Irreducibility.lean`:

* `Berlekamp.checkPowChainLinearIncrementalStep` — single-step
  `powChain[k+1] == (powChain[k])^p mod f` check.
* `Berlekamp.checkPowChainLinearIncremental` — replaces the
  `frobeniusXPowModLinear`-driven from-scratch chain reconstruction with
  a base-case (`powChain[0] = X mod f`) plus `n` incremental steps.
  Total work drops from `O(Σ p^k)` to `O(n · p)`.
* `Berlekamp.checkIrreducibilityCertificateLinearIncremental` — same
  certificate-shape entry point as the existing linear checker.
* `checkPowChainLinearIncremental_spec` — induction on `k` using
  `FpPoly.frobeniusXPowMod_zero` for the base and
  `FpPoly.powModMonicLinear_eq_powModMonic` plus
  `FpPoly.frobeniusXPowMod_succ` for the step.
* `checkIrreducibilityCertificateLinearIncremental_rabinTest` —
  soundness bridge mirroring the existing `_Linear_rabinTest` proof but
  using the incremental spec.

`HexGfqField/EmitFixtures.lean`:

* `m_p5_n6_certificate` (Conway poly for `GF(15625)`) and
  `m_p7_n6_certificate` (Conway poly for `GF(7^6)`) as record literals.
* Per-modulus `_certificate_check` proofs via `simp + constructor` chain
  (one extra inner `constructor` vs. the linear-cert path because the
  incremental check adds a `powChain[0] = X mod f` conjunct).
* Per-modulus `_irr` theorems route through
  `Berlekamp.rabinTest_imp_irreducible ∘
   checkIrreducibilityCertificateLinearIncremental_rabinTest`.
