# Strassen-Winograd SPEC (#8617)

## Accomplished
Wrote the SPEC for Strassen-Winograd matrix multiplication as a section of
`HexMatrix/SPEC/hex-matrix.md` (the publish step copies only that file, per
`scripts/release/sync_released.py`, so a standalone `strassen-winograd.md`
would not reach the released repo). The section covers all four issue bullets:
benchmarks (measured crossover cutoff, `n^{log₂ 7}` cost model, structural-layer
comparator absence), a pluggable-but-not-default base kernel (`StrassenConfig`
with a separate `Valid` predicate), copy avoidance (block views inside the
recursion, materialize only at the base case, flat-backing as follow-up), and
the literature (Strassen 1969, Boyer-Dumas-Pernet-Zhou ISSAC 2009 memory
schedule, CoqEAL prior art). Also updated the `mul` docstring in
`HexMatrix/Basic.lean` to point at the SPEC and record the `[Sub R]` split, and
noted the new Strassen bench target's comparator classification in the
"External comparators" section.

Ran `/second-opinion` (Codex) on the draft. Codex confirmed the S/T/P/U
schedule is algebraically correct and flagged real defects, which I fixed:
- **Critical**: the odd-dimension `⌊n/2⌋`/`⌈n/2⌉` half-split does not type-check
  (`A₂₁ + A₂₂` mismatches shapes). Replaced with "even dimensions required to
  split, odd dimensions padded to even per level", recursing on rectangular
  even blocks, with static power-of-two padding and dynamic peeling as
  documented alternatives.
- **High**: the correctness theorem needs lawful ring axioms. Split the def
  (`[Mul][Add][Sub][OfNat 0]`) from the proof (`[Lean.Grind.Ring R]`, the class
  the existing laws in `MatrixAlgebra.lean` use).
- **High/Medium**: `StrassenConfig` bundled a proof field referencing `mul`
  without instances. Made it data-only with instance params and a separate
  `StrassenConfig.Valid` predicate.
- **Medium**: made the "default at runtime" mechanism explicit (csimp on
  `mulStrassen`'s own reference form plus a mechanical caller switch; generic
  `*` stays naive), dropped the wrong "`[Sub R]` equivalently `[Neg R]`", and
  clarified views-vs-base-kernel (materialize at leaves) and logical-vs-storage
  schedule.

## Current frontier
The SPEC is complete and internally consistent. No Lean implementation was
written: per design principle 7 (no scaffolding defs) and the issue title
("write a spec"), the deliverable is the specification. `HexMatrix` builds
green (only docstring text changed in source).
