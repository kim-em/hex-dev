### What landed

Two additions to `HexBerlekampZassenhausMathlib/IntReductionMod.lean`:

1. `dvd_gcd_mul_gcd_of_dvd_mul` (`private`): GCDMonoid helper proving
   `g ∣ x * y → g ∣ gcd g x * gcd g y` by two applications of Mathlib's
   `dvd_gcd_mul_of_dvd_mul`. Stated for `EuclideanDomain α + DecidableEq α`
   and discharged via `letI := EuclideanDomain.gcdMonoid α` so the
   `EuclideanDomain.gcd` in the conclusion unifies with the `GCDMonoid.gcd`
   used by the Mathlib lemma.

2. `Polynomial.gcd_derivative_associated_divRadical_of_charZero`: the main
   substrate fact. Proof by `UniqueFactorizationMonoid.induction_on_coprime`
   over four cases:
   * `h0` (`p = 0`) and `h1` (`IsUnit p`): both sides reduce to the input
     via `radical_zero`, `radical_of_isUnit`, `derivative_zero` /
     `derivative_C`, and the appropriate `gcd_zero_*` lemmas.
   * `hpr 0` (`p^0 = 1`): reduces to `Associated 1 1`.
   * `hpr (k+1)`: two sub-goals.
     - `divRadical (q^(k+1)) ~ q^k`: from `radical_pow_of_prime`
       (`= normalize q`) + `radical_mul_divRadical` (`normalize q *
       divRadical (q^(k+1)) = q^(k+1) = q * q^k`), cancel `normalize q` via
       `Associated.of_mul_left` against `normalize_associated q`.
     - `gcd q^(k+1) (derivative q^(k+1)) ~ q^k`: combine
       `q^k ∣ g` (`dvd_gcd` plus `pow_dvd_pow` and
       `derivative_pow_succ`'s explicit `C(k+1) * q^k * derivative q` form)
       and `g ∣ q^k` (extract `m ≤ k+1` from `dvd_prime_pow`; rule out
       `m = k+1` by chasing `q ∣ derivative q`, then use char-0 to make
       `C(k+1)` a unit, then `PerfectField.ofCharZero` →
       `PerfectField.separable_of_irreducible` → `IsCoprime q (derivative q)`
       → `IsCoprime.isUnit_of_dvd' dvd_rfl` contradicting
       `Prime.not_unit`), then `associated_of_dvd_dvd`.
   * `hcp` (`IsRelPrime x y` with IHs): forward via
     `divRadical_dvd_self` + `divRadical_dvd_derivative` + `dvd_gcd`;
     reverse via `divRadical_mul` (`IsRelPrime.isCoprime` in the PID
     `K[X]` then `divRadical_mul`), `dvd_gcd_mul_gcd_of_dvd_mul` to factor
     `g ∣ xy` into `g ∣ gx * gy`, and per-component chain
     `gx ∣ gcd x x' ∣ divRadical x` (and the y-symmetric form). The
     component chain uses `Polynomial.derivative_mul` followed by
     `dvd_add_left`/`dvd_add_right` to extract `gx ∣ derivative x * y`
     from `gx ∣ derivative (xy)`, then
     `IsCoprime.of_isCoprime_of_dvd_left` and
     `IsCoprime.dvd_of_dvd_mul_right` to peel the `* y` factor.

### Scoping correction

The original issue body identified an "algebraic bottleneck" in case 5
(`hcp`): the claimed missing UFD-coprime divisor-decomposition lemma
`g ∣ xy → g ∣ gcd g x * gcd g y` (under `IsCoprime x y`). In fact
Mathlib's `dvd_gcd_mul_of_dvd_mul` (`Algebra/GCDMonoid/Basic.lean:478`)
gives `k ∣ m * n → k ∣ gcd k m * n` in any `GCDMonoid` — applied twice
via `mul_comm`, this yields the supposedly-missing helper without
requiring coprimality at all. The decomposition path (`path A`) and
alternative multiplicity path (`path B`) the issue body suggested both
turn out to be unnecessary detours.
