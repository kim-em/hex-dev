# 2026-05-04T07:47:19Z — feature session (652f4584)

## Accomplished

Wired up a python-flint oracle for `HexGF2` (closes #2000), matching
the pattern already in place for `HexPoly`, `HexBerlekamp`,
`HexBerlekampZassenhaus`, and `HexMatrix`.

Two families of fixtures, one driver per:

* **`F_2[x]` arithmetic** — `mul`, `gcd`, `divmod` on polynomial
  pairs at degrees 16, 32, 64, 128.  Sources are standard CRC
  generator polynomials (`CRC-16-CCITT`, `CRC-32-IEEE`, `CRC-64-ECMA`)
  plus the GHASH polynomial at degree 128, paired with reflected /
  high-degree partners so each case spans at least one packed-word
  boundary.
* **`GF(2^n)` field arithmetic** — `gf_mul`, `gf_inverse`,
  `gf_frobenius` (squaring in characteristic 2) at `n ∈ {4, 8, 16}`.
  The `n = 8` case uses the AES irreducible `x^8 + x^4 + x^3 + x + 1`,
  with the classical `0x53 ↔ 0xCA` test pair so the oracle also
  incidentally cross-checks the AES inverse table; `n = 4` and
  `n = 16` use `x^4 + x + 1` and `x^16 + x^12 + x^3 + x + 1`.

Files added/modified:

* `HexGF2/EmitFixtures.lean` — JSONL emitter; emits the modulus and
  inputs as `poly` records with `modulus = 2`, then one `result` per
  cross-checked op.  Irreducibility witnesses are local `sorry` axioms
  (same convention as `HexGF2/Conformance.lean`).
* `scripts/oracle/gf2_flint.py` — replays each `F_2[x]` op via
  `flint.nmod_poly(_, 2)` and each `GF(2^n)` op via
  `(a * b) % modulus` and `nmod_poly.xgcd` for inversion.  Mirrors the
  `0⁻¹ = 0` Lean junk-value convention so a deliberate corruption
  surfaces as a normal coefficient diff.
* `conformance-fixtures/HexGF2/gf2.jsonl` — 38-line committed sample
  replayable via `python3 scripts/oracle/gf2_flint.py --check`.
* `.github/workflows/conformance.yml` — adds the
  `hexgf2_emit_fixtures` build + diff + oracle run to the
  `oracle-pyflint` job.
* `lakefile.lean` — new `lean_exe hexgf2_emit_fixtures` target.

Verification locally:

* `lake build hexgf2_emit_fixtures` green (3 sorry warnings only,
  matching the `Conformance.lean` AES axiom convention).
* `lake exe hexgf2_emit_fixtures` deterministic against the
  committed fixture (`diff -u` exit 0).
* `python3 scripts/oracle/gf2_flint.py --check` reports
  `checked 21 case(s), 0 failure(s)` against python-flint 0.8.0 (4
  `F_2[x]` cases × 3 ops + 3 `GF(2^n)` cases × 3 ops = 21).
* Deliberate corruptions (flipped low bit of `f2x/deg64` mul,
  bogus AES inverse `[0,…,0,1]`) trip the oracle with precise
  coefficient diffs and exit code 1.

## Current frontier

`HexGF2` was previously the largest `done_through ≥ 4` library
without an independent oracle; this PR closes that gap.  The G3
(`HexGF2` extern vs pure-Lean clmul) cross-check landed in #2008 and
remains complementary — this PR is the python-flint independent
oracle.

## Next step

(#2004).  The `HexGfq` packed-vs-generic cross-check (#2033) already
