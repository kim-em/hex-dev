## Sign-premise correction (the caveat #7365 flagged)

`Hex.ZPoly.primitivePart` divides by the **nonnegative** content but does
**not** sign-normalize (verified against `DensePoly.primitivePart`;
`primitivePart_eq_self_of_primitive` holds for any primitive polynomial,
any sign). Hence
`primitivePart (dilate lc g) = sign(lc)^deg * sign(lf) * factor`.

`normalizeFactorSign factor = factor` only forces `0 < leadingCoeff factor`;
it does **not** control `sign(lc)`. So the issue's statement is **false as
written** — counterexample `core = -X^2 + 1`, `factor = X - 1` satisfies
all stated hypotheses but yields `primitivePart (dilate lc g) = -factor`.

Per the issue's own instruction ("the correct extra hypothesis ... must be
threaded — report the exact shape on this issue"), the landed signature
adds `hlc : 0 < leadingCoeff core` (mathematically necessary) and
`hfdeg : 1 ≤ factor.degree?.getD 0` (excludes the degenerate unit-factor
case; automatic for the irreducible factors #7364 feeds). A comment with
this analysis is posted on #7365.
