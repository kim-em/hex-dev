## Accomplished

Salvaged PR #2789 (DDF saturation lemma for square-free residuals).
Conformance CI was failing on `HexBerlekampZassenhausMathlib/Resultant.lean`
at three `lia` calls (lines 123, 131, 134) reporting `grind` failure.

Root cause: the new `CommRingDerivative` section in `HexPoly/Euclid.lean`
contained a global `attribute [instance 1100] Lean.Grind.Semiring.natCast`,
raising the `NatCast` projection priority for every downstream importer.
That changed the cast paths visible to `lia`/`grind` in
`HexBerlekampZassenhausMathlib/Resultant.lean`, which transitively imports
`HexPoly.Euclid` via `HexPolyZMathlib.Mignotte → HexPolyZMathlib.Basic →
HexPolyMathlib.Basic → HexPoly → HexPoly.Euclid`. The grind diagnostics
showed nat-to-int coercions appearing inside what used to be pure `Nat`
goals.

Fix: scoped the priority bump with `local` (`attribute [local instance 1100]`)
so it only applies inside `HexPoly/Euclid.lean` where the new
`derivative_mul` proofs need it. `lake build HexPoly.Euclid` and the full
conformance target list both build clean.
