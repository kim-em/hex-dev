  `mem_T_iff_exists_irreducibleFactor_representingSubset_of_scaledRecombinationCandidate_of_primitive_pos_lc_core`

    the same `conv_lhs => rw [hr₂, hr₁]` LHS-only rewrite to dodge

    argument on the RHS),
  - delegates into the new `_of_bound` sibling with
    `B' := defaultFactorCoeffBound core`, discharging `hvalid` for an
    arbitrary normalised factor `g` by lifting to `g ∣ core` via the
    `g ∣ cand ∣ target ∣ core` chain and invoking
    `defaultFactorCoeffBound_valid core hcore_ne g hg_dvd_core`.

## Layout note

The convention established by the substrate `_of_bound` pairs
(`exists_representingSubset_*_of_bound` then wrapper at lines 11041 /
11178; `exists_mem_representedSubset_*_of_bound` then wrapper at 11290
/ 11397) places the `_of_bound` *before* the wrapper in the file —
necessary because Lean 4 top-level definitions are sequential and the
wrapper forward-references the `_of_bound`. The issue text said
"immediately following" but the precedent and dependency direction
both require the `_of_bound` to come first. Followed the established
layout: `_of_bound` inserted at line ~11476 (replacing the original's
doc comment region), with the wrapper now occupying the next block
with an updated doc comment that describes the wrapper-of-`_of_bound`
relationship.
