# 2026-05-04 05:38Z — feature session, issue #1988

## Accomplished

- Closed #1988 by adding the `HexPolyZ` python-flint oracle layer.
- New `HexPolyZ/EmitFixtures.lean` (lake exe `hexpolyz_emit_fixtures`)
  emits JSONL fixtures + Lean-side result records for:
  - `content` / `primitive_part` over degree-8 and degree-12
    polynomials with content up to 30, plus an already-primitive
    degree-8 case and the all-zero edge case.
  - `gcd_z` over Z on degree-6 and degree-10 pairs sharing a degree-4
    factor.  The integer gcd associate is computed as
    `ratPolyPrimitivePart (DensePoly.gcd (toRatPoly f) (toRatPoly g))`
    so its sign convention matches `fmpz_poly.gcd`.
  - `divmod` over Z at dividend degree 8 (one quadratic divisor, one
    monic-quartic divisor), restricted to exact division — the regime
    where Lean's truncating `Int` divMod and `fmpz_poly` agree.
  - `mignotte_coeff_bound`: emits `[k, j, bound]` triples for several
    factor-degree / coefficient-index pairs at degrees 8 and 12.
- New `scripts/oracle/polyz_flint.py` re-runs each operation through
  python-flint and re-derives the Mignotte bound from raw coefficients
  via independently-implemented `binom` and `ceil_sqrt` helpers.  Note
  python-flint exposes `content` and `gcd` directly but no
  `primitive_part`, so we use the standard `f // f.content()` idiom.
  Used `_height` is *not* the right witness for the Mignotte L2 bound
  (the issue mentioned it incidentally) — independent recomputation
  gives a stronger cross-check and matches `coeffL2NormBound`'s
  `ceilSqrt(∑ cᵢ²)` exactly.
- Committed the regression sample at
  `conformance-fixtures/HexPolyZ/polyz.jsonl`; the CI step `diff -u`s
  fresh emission against it.  20 result records across the four
  operations.
- Extended `.github/workflows/conformance.yml`'s `oracle-pyflint` job
  with HexPolyZ build + cross-check + oracle steps alongside the
  existing HexPoly steps.  Failure-record artefact upload is shared.
- Verified locally with python-flint 0.8.0: `polyz_flint.py --check`
  exercises 20 cases with 0 failures; deliberately corrupting one
  `primitive_part` value in the JSONL trips a `FAIL` and exit 1 (then
  reverted).  HexPoly oracle still passes (6 cases).
- `lake build hexpolyz_emit_fixtures` and `lake build HexPolyZ` are
  green; pre-existing `HexPolyZ.Basic` sorries are unchanged.
