# 2026-05-04 06:01Z — feature session, issue #2018

## Accomplished

- Bumped `HexBerlekamp/Conformance.lean` typical-case fixture from
  `mixedDegreePoly` (degree 3) to `bigPoly` (degree 8) over `F_5`.
- `bigPoly = (1 + x) * (2 + x²) * (x⁵ - x - 1)`, with the quintic factor
  `x⁵ - x - 1` irreducible by Artin-Schreier (`a ≠ 0`).
- Stored `bigPoly` in expanded form `[3, 1, 2, 3, 4, 2, 2, 1, 1]` rather
  than as the symbolic three-way product so the monic proof reduces by
  `rfl` without exhausting `maxRecDepth`. A `set_option maxRecDepth 2048`
  guard checks `bigPoly == linearPoly * irreducibleQuad * irreducibleQuint`.
- Added degree-8 fixtures:
  - `basisSize = 8` and `coeffVector` literal
  - `F = Q - I` property over the full 8×8 fixed-space matrix
  - `frobeniusDiffMod bigPoly _ 1` literal (degree 5 < 8 ⇒ no reduction)
  - `splitFactorAt bigPoly X 4 = [1, 1]` (the only `F_5` root of bigPoly is `x = 4`)
  - `kernelWitnessSplit?` reconstructive property
  - DDF result has bucket degrees `[1, 2, 5]`, product equals `bigPoly`,
    residual is unit
- Kept the existing edge-case fixtures (`unitPoly`, `linearPoly`,
  `irreducibleQuad`, `reducibleQuad`) and their literal tests intact.
- Replaced the literal-coefficient DDF tests with degree + product +
  residual-unit property checks, since `DensePoly.gcd` (raw EEA) returns
  the gcd up to a unit scalar — the EEA on degree-8 inputs happens to
  terminate with a non-monic representative (e.g., `4 + 4x` for the
  linear bucket).
- Verified deliberate corruption of `coeffVector bigPoly bigPoly` from
  `[3, 1, 2, 3, 4, 2, 2, 1]` to `[3, 1, 2, 3, 4, 2, 2, 2]` trips the
  build, then reverted.
- `lake build HexBerlekamp.Conformance` clean at ~1.0-1.2s elaboration
  (well within the 2s soft budget).
