Resolved issue #2003: "HexGfqField: python-flint oracle for F_q field
ops + Frobenius".

- Added `gfqfield` JSONL fixture kind to `scripts/oracle/common.py`
  (schema: `p`, `modulus`, `a`, `b`, `zexp`).
- Added `Hex.Conformance.Emit.emitGfqFieldFixture` for the Lean side.
- Created `HexGfqField/EmitFixtures.lean` (`hexgfqfield_emit_fixtures`
  exe in `lakefile.lean`).  Sixteen cases — every `(p, n)` with
  `p ∈ {2, 3, 5, 7}` and `n ∈ {2, 3, 4, 6}` — emit one fixture record
  plus five op results (`mul`, `inv`, `div`, `frob`, `zpow`).  Each
  modulus is constructed as a struct literal so `decide` can discharge
  the `0 < degree m` obligation without recursing through
  `trimTrailingZeros`; irreducibility is taken as a per-modulus axiom
  (matches the `axiom modulus_irreducible` pattern in
  `HexGfqField/Conformance.lean`).  All sixteen moduli were
  independently verified irreducible by python-flint at design time.
- Created `scripts/oracle/gfqfield_flint.py` cross-checking each op
  through `flint.fq_default_ctx(modulus=fmpz_mod_poly(m, p))`.
- Committed `conformance-fixtures/HexGfqField/gfqfield.jsonl` (96
  records) as the deterministic CI sample.
- Wired the `oracle-pyflint` job in `.github/workflows/conformance.yml`
  to (1) diff the freshly-emitted JSONL against the committed sample
  and (2) run the python-flint oracle on the live emission.
