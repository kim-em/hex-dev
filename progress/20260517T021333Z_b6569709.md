### Mathlib-bridge side (`HexBerlekampZassenhausMathlib/Basic.lean`)

Added `factorsModP_natDegree_pos_of_factorsModPBerlekampForm` adjacent
to the `Nodup` sibling (around lines 4082–4244):

```lean
theorem factorsModP_natDegree_pos_of_factorsModPBerlekampForm
    (f : Hex.ZPoly) (data : Hex.PrimeChoiceData)
    (hform : Hex.factorsModPBerlekampForm f data)
    (hgood : letI := data.bounds; Hex.isGoodPrime f data.p = true)
    (hf_pos : 0 < f.degree?.getD 0) :
    letI := data.bounds
    ∀ g ∈ data.factorsModP,
      0 < (HexPolyZMathlib.toPolynomial (Hex.FpPoly.liftToZ g)).natDegree
```

Proof shape (mirroring the Nodup sibling):

- Destructure `hform` to view `data.factorsModP` as the Berlekamp factor
  array of `monicModularImage (modP data.p f)`.
- Discharge positivity of `monicModularImage (modP data.p f)` from the
  input degree + `isGoodPrime`'s leading-coefficient admissibility:
  `hf_pos` gives `f.size ≥ 2`; `isGoodPrime_leadingCoeffAdmissible`
  forces `(modP data.p f).coeff (f.size - 1) ≠ 0`, hence
  `(modP data.p f).size = f.size ≥ 2`; `monicModularImage` is a
  nonzero scale, preserving size via
  `Hex.FpPoly.scale_size_eq_of_ne_zero`.
- Apply `berlekampFactor_factors_pos_degree` (the new Mathlib-free
  lemma) to get per-factor positivity on the Berlekamp output.
- Bridge per-factor `0 < g.degree?.getD 0` (FpPoly side) to
  `0 < (HexPolyZMathlib.toPolynomial (liftToZ g)).natDegree` (integer
  side) by:
  * deriving `(liftToZ g).size = g.size` inline (leading coefficient
    survives the lift because its `toNat` representative is positive),
    hence `(liftToZ g).degree? = g.degree?`;
  * applying `HexPolyMathlib.natDegree_toPolynomial`, which gives
    `(toPolynomial p).natDegree = p.degree?.getD 0` unconditionally
    (no monicness or leading-coefficient hypothesis required).
