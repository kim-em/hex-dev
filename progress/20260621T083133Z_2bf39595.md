# F_p Berlekamp EDF split-loop guard (#8274)

Session type: feature (one-shot via `pod once 8274`). UTC 2026-06-21T08:31.

## Accomplished

Landed the value-preserving constant-residue guard on the F_p Berlekamp
equal-degree split loop ÔÇö the bottleneck localized by #8270 (#8267).

**The fix.** `kernelWitnessSplit? f witness` now short-circuits to `none`
when `(witness % f).size ÔëĄ 1` (the witness is constant modulo `f`), skipping
the full `p`-wide `gcd(f, witness Ôłĺ c)` constant sweep. Once a factor is
irreducible, *every* kernel witness is constant modulo it, so the sweep is
pure waste ÔÇö the `╬ś(r┬▓┬Ěp)`-gcd tax the localization report measured.

**Value-preserving, proved (no sorry/axiom).** The guard is justified by
`isNontrivialSplitFactor_false_of_mod_size_le_one`: over a field, a witness
constant mod `f` yields no nontrivial split ÔÇö each `gcd(f, witness Ôłĺ c)` is
an associate of `f` (when `c` hits the residue) or a nonzero constant
otherwise. The supporting `kernelWitnessSplitAux_none_of_mod_size_le_one`
proves the skipped sweep would have returned `none`, so `kernelWitnessSplit?`
is extensionally equal to the unguarded `kernelWitnessSplitAux` and every
existing soundness lemma (`RabinSoundness`, completeness, `Nodup`) carries
over. The defs keep `[ZMod64.Bounds p]` (the guard's `%`/`size` need only
that); the new soundness lemmas carry `[ZMod64.PrimeModulus p]`.

Proof key pieces: `div_mul_add_mod` to reduce `witness Ôłĺ c` by `(witness/f)┬Ěf`
onto `(witness%f) Ôłĺ c`; `dvd_sub_poly`/`dvd_mul_left_poly` for the gcd
divides the residue; `degree?_mul_eq_add_degree?` for the unit case;
in-file `size_eq_of_dvd_dvd` for the associate case.

## Notes on the directive premise

Deliverable 2 asked to "replace" the loop with randomized CantorÔÇôZassenhaus
modular-exponentiation EDF. That would discard the entire deterministic-
Berlekamp soundness infrastructure (product/`Nodup`/completeness, ~1700 lines)
and re-prove it for a randomized algorithm ÔÇö a multi-session rewrite. The
constant-residue guard captures the same asymptotic win (the futile-gcd tax)
while staying value-preserving and keeping every proof intact, which is what
deliverable 3 explicitly sanctions ("value-preserving; any `@[csimp]` swap
proved equivalent"). Deliverable 1 (per-iteration profiling) was already done
in the #8270 localization report; this session executes the fix.

## Current frontier / Next step

The guard removes the futile-sweep tax. Any further F_p speedup would now come
from the genuine algorithmic upgrade (CantorÔÇôZassenhaus with `x^((p^dÔłĺ1)/2)`
modexp), which is a larger, soundness-infrastructure-replacing effort and a
candidate for a separate, scoped issue if high-degree F_p factorization
remains on the critical path.
