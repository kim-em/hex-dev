## Key findings (de-risk the remaining atomic migration)

The two prior sessions inlined `(normalizeMonic (DensePoly.gcd …)).2` and
measured 42/42 file-wide errors incl. 8 hard kernel timeouts, declaring it a
larger-than-one-session atomic change. I reproduced and resolved the timeout
mechanism:

1. **`@[irreducible] monicGcd` eliminates the 3000-range timeout cluster.**
   With the named def swapped into the 6 executable sites, a plain
   (semireducible) def still timed out at the same sites because
   `simp`/`decide`/defeq unfolds it into `normalizeMonic`. Marking it
   irreducible removed the `:3082/:3108/:3139` timeouts entirely (measured:
   8 → 5 timeouts).
2. **The remaining 5 timeouts (`:7652–:7934`) collapse via one invariance
   lemma.** They live in the `yunContribution_scalarSync` family, which
   exists *only* to track the unit ambiguity between `gcd (C u_c·c)(C u_w·w)`
   and `gcd c w`. `monicGcd` normalizes that unit away, so
   `monicGcd (C u_c·c)(C u_w·w) = monicGcd c w`. Proving that one
   `monicGcd_scalar_invariant` lemma (from the existing `gcd`-scaling lemmas
   at ~`:1699–:1880` plus `normalizeMonic (C v · g) = normalizeMonic g` for
   nonzero `v`) makes the scalar-sync proofs near-trivial instead of
   timing-out.
3. **Providers become provable, not just removable.** With monic recursion
   variables, `squareFreeContributionReachable` (`f.size = 1 → f = 1`) holds
   by `squareFreeContributionReachable_of_monic`; monicity propagates through
   `monic_div_monicGcd_left/right_of_monic`. So
   `YunDerivativeActiveRawStateProvider` turns into a lemma and the
   `hrawState` hypothesis can be discharged from the public chain
   (`…_pow_obligation` → `_correct_pow_of_nonzero` → `_correct` →
   `squareFreeAux_weightedProduct_nonzero` → `squareFree_weightedProduct`).

Measured error taxonomy for the full def-swap (irreducible def): ~70 errors,
of which ~55 are mechanical `let y := DensePoly.gcd c w` → `monicGcd c w`
proof-mirror renames (+ swap `gcd_dvd_left/right`/`div_gcd_*_reconstruct` →
the `monicGcd_*` analogues, which now also take `hp`), 5 scalar-sync
timeouts (finding 2), 2 kernel-unknown and 2 rewrite failures clustered with
the providers, and the provider re-grounding.
