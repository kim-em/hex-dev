# Mathlib-free small-mod singleton branch composition

Closes #4885.

## Accomplished

Added the Mathlib-free branch-level composition theorem
`Hex.BerlekampZassenhaus.SmallModSingleton.squareFreeCore_irreducible_of_small_mod_singleton`
to `HexBerlekampZassenhaus/SmallModSingleton.lean`. From the small-mod
singleton branch's `choosePrimeData?` and `factorsModP.size Ōēż 1`
witnesses plus a non-constancy marker, the theorem concludes
`Hex.ZPoly.Irreducible` for the square-free core of
`Hex.normalizeForFactor f`.

The proof composes:

1. `monicModularImage_modP_irreducible_of_choosePrimeData?_small` (#4858,
   already in the same file) ŌĆö `FpPoly.Irreducible (monicModularImage
   (modP p core))` from the singleton-bounded Berlekamp count;
2. `FpPoly.irreducible_of_scale_of_ne_zero` applied to the unit-scaling
   equation `monicModularImage r = scale (leadingCoeff r)Ōü╗┬╣ r` to
   transfer to `FpPoly.Irreducible (modP p core)`;
3. `Hex.ZPoly.Irreducible_of_modP_irreducible_of_primitive_of_admissible`
   (#4884, just landed on `main`) ŌĆö the Mathlib-free Gauss reduction-mod-`p`
   transfer.

Side conditions are discharged from the existing executable invariants:

* `ZPoly.Primitive core` via
  `primitiveSquareFreeDecomposition_squareFreeCore_repeatedPart_primitive`
  on `(extractXPower (primitivePart f)).core`, plus a small private
  helper `ZPoly_Primitive_left_of_mul` (primitivity is closed under left
  factor of a primitive product, since `content (p*q) = content p *
  content q` with both contents non-negative and product `1` forces
  both to be `1`).
* `leadingCoeffAdmissible core primeData.p` via
  `choosePrimeData?_isGoodPrime` and `isGoodPrime_leadingCoeffAdmissible`.
* `1 < core.size` (non-constancy precondition of the Gauss transfer)
  from `hdeg : core.degree?.getD 0 ŌēĀ 0`.

The proof factors through a private intermediate
`zpoly_irreducible_of_choosePrimeData?_small_of_primitive` that
generalises the chain to any primitive `ZPoly` with `1 < size`, so the
`normalizeForFactor` specialisation is a thin wrapper.

### Signature deviation

The issue's deliverable spec did not include `hdeg`, but flagged
"(or an equivalent shape consumable by the #4825 capstone)" as
permissible. The `hdeg` hypothesis is **necessary**: without
`1 < core.size`, the conclusion `Irreducible core` is false when
`core = C ┬▒1` (primitive constants survive `choosePrimeData?` and
the singleton Berlekamp bound vacuously). The capstone consumer
already carries `hdeg` for branch dispatch ŌĆö the Mathlib-side analog
`factor_small_mod_singleton_branch_entry_irreducible_of_choosePrimeData`
(`HexBerlekampZassenhausMathlib/IntReductionMod.lean:2522`) threads it
explicitly ŌĆö so consuming it here imposes no new constraint on callers.

### Mathlib-free tactics

`set`, `push_neg`, and `by_contra` from Mathlib are unavailable in
`HexBerlekampZassenhaus/SmallModSingleton.lean`. Replaced with plain
`let`-bindings, `rcases Nat.lt_or_ge`, and `rcases Nat.eq_zero_or_pos`
respectively. (An earlier attempt with `let core := ...` triggered a
heartbeat timeout during `whnf` typeclass synthesis on the outer
`Bounds` instance; factoring the proof through a helper that takes
`primeData` as a parameter resolved the timeout.)
