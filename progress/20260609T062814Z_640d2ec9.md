## Key finding: directive's premise underestimates difficulty

The issue body's hint "Track the `sqrtAux` Newton-iteration
termination invariant ... compose with the existing fuel bound +
initial-condition (`x = n`) check" understates the work:

- The stop-condition lemma (`sqrtStep n x ≥ x → x * x ≤ n` for
  `x > 0`) is straightforward (~10 lines).
- But proving the iteration **terminates via stop** within the fuel
  budget (rather than exhausting fuel) requires showing Newton
  convergence on Nat from `x = n` lands at floor sqrt within
  `2 * n.log2 + 1` steps.
- `Nat.log2` does **not** strictly decrease per Newton step: for
  example `x = 7, n = 8` gives `sqrtStep = 4`, both `log2 = 2`. So
  naive strong induction on `x.log2` (the obvious "compose" approach)
  does not close.
- A valid invariant must absorb per-step log2 stagnation. Empirically
  `x_k ≤ n / 2^k + 2k` holds, but proving it requires a careful
  Newton-step Nat-arithmetic bound. There is no `Batteries`/`Mathlib`
  inheritance path (file is `Std`-only).
