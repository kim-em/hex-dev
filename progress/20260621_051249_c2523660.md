# Localize the F_p Berlekamp bottleneck (#8267)

Session type: feature (one-shot `pod once 8267`). UTC 2026-06-21T05:12.

## Accomplished

Rigorously localized the ~90% `factorFast` cost that #8267 / #8062 item 4
attribute to `choosePrimeData?`, and **refuted the issue's deliverable-2
hypothesis** that the Berlekamp matrix build + nullspace (Frobenius /
Q-matrix) is the bottleneck.

Measured per-stage breakdown (deg-12 input, p=13), via a standalone
profiling exe that defeats CSE by indexing a distinct input per timed
iteration:

- `factorFast` ~86 ms; `choosePrimeData?` ~74 ms (~86%).
- `isGoodPrime` over all scanned primes: ~0.09 ms (0.1%).
- Berlekamp matrix + nullspace (`fixedSpaceKernel`): ~1.76 ms (2.3%).
- Berlekamp factorization (`berlekampFactor`): ~76 ms; of which the
  **equal-degree split loop ≈ 74 ms (97.7%)** — the real bottleneck.

Root cause: `kernelWitnessSplit?` runs a full p-constant
`gcd(factor, witness − c)` sweep for every (factor, witness) pair,
including the `Θ(r²·p)` *futile* pairs where the witness is constant mod
the factor (i.e. every witness once a factor is irreducible).

Validated the fix (prototyped in the throwaway harness, not the library):
a guard skipping the sweep when `witness mod factor` is constant takes
`berlekampFactor` from ~76 ms to ~6 ms (**12.7×**) with **0 factor-multiset
mismatches over 32 distinct inputs**.

## Key decisions / patterns

- **Defeat CSE in micro-benchmarks.** A fixed-input timing loop collapsed
  `berlekampFactor` to 37 ns (compiler computed it once). Indexing a
  distinct input per iteration restored honest 76 ms numbers. Any future
  in-`#eval`/exe perf probe in this repo must vary the input per iter.
- **The fix proof is real work, not a one-liner.** It touches load-bearing
  lemmas consumed by `RabinSoundness.lean`, needs a completeness lemma,
  and that needs ~4 missing FpPoly dvd/size helpers (built on the existing
  `size_mul_eq_add_sub_one`). Deferred to #8269 rather than risk a broken
  CI-gated `HexBerlekampZassenhausMathlib` in a non-interactive one-shot.
- **Profiling exe was scratch**, removed before commit (lakefile reverted).
