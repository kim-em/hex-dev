# 2026-06-02T02:45Z — implement the inverted-dependency guard

Closes #6209.

## Accomplished

- Factored the import-DAG reachability check out of `check_dag.py` into a
  reusable `may_import(l_a, l_b, libraries, closure=None)` predicate in
  `scripts/libgraph.py`. Predicate is pure: true iff `l_b == l_a` or
  `l_b` is in `l_a`'s transitive dependency closure. Accepts an optional
  precomputed closure for callers (like `check_dag.py`) that make many
  calls in a row.
- Rewired the import-boundary check in `check_dag.py` to call
  `may_import` with the closure it already computes. Behaviour-preserving:
  `python3 scripts/check_dag.py` still exits 0.
- Added `scripts/maintenance/scrub_inverted_deps.py`, the standalone
  backlog scrub the housekeeping loop runs. For each open issue it parses
  the `library:` line and every `depends-on: #N`, resolves each
  dependency's `library:` line (cached), and flags edges where
  `not may_import(L_A, L_B)`. For a flagged edge under `--apply` it drops
  the line from the body, posts a comment, removes the `blocked` label
  if no other open real dependency remains, and applies `replan`. Defaults
  to dry-run; only `--apply` mutates GitHub. Issues missing a `library:`
  line (carrier or dep) are reported, never scrubbed.

## Current frontier

Running the dry-run today reports zero inverted edges and 26 issues with
no `library:` line (PR #6203 introduced the convention but did not

becomes effective as planners start writing `library:` lines into new
issues, and as the dev-pod `add depends-on` path adds the reject-at-
creation guard (out of repo, separate follow-up noted in the issue).
