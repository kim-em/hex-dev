# Cantor–Zassenhaus no-rescan split for berlekampFactor (#8281)

Session type: feature (one-shot `/once`). UTC 2026-06-21T11:02.

## Accomplished

Removed the residual quadratic-in-(factor count) tax in `berlekampFactor`
(`HexBerlekamp/Factor.lean`): the former `berlekampFactorLoop` restarted
`splitFirstFactor?` from the front of the factor list on every pass,
re-testing every already-irreducible factor against every kernel witness.
Replaced it with a recursive `fullySplit` that splits each factor fully
(depth-first, factor-before-cofactor) and emits an irreducible factor as a
leaf the instant `splitWithWitnesses?` reports `none` — so each factor is
tested against the witnesses exactly once.

**Deliverable 1 — restructure (shape (a)).** New `private def fullySplit`
recurses on `split.factor ++ split.cofactor`, fuel-bounded by `f.size + 1`
(`size` strictly decreases at every split). `berlekampFactor` now calls
`fullySplit witnesses (f.size + 1) f`. The output preserves the former loop's
exact order (both are the pre-order leaf listing of the same deterministic
split tree), so the factor multiset *and order* are unchanged. Deleted the
whole loop infrastructure: `splitFirstFactor?`, `berlekampFactorLoop`, and
their now-dead helper lemmas (`*_length_succ_of_some`, `*_length_ge`,
`*_singleton_none_iff`, `*_invariant`, `*_pos_invariant`,
`*_terminates_or_full`, `factorProduct_length_le_degree`,
`splitWithWitnesses?_none_of_splitFirstFactor?_none`). None were referenced
outside this file.

**Deliverable 2 — re-proved structural lemmas on the tree shape.** The
per-step `splitWithWitnesses?_*` lemmas (product, factor/cofactor positivity,
size descent) are reused unchanged; added `ne_zero_of_pos_degree`,
`size_ge_two_of_pos_degree`, `splitWithWitnesses?_cofactor_size_lt`. New
tree-recursion lemmas: `factorProduct_append`, `factorProduct_fullySplit`,
`fullySplit_ne_nil`, `fullySplit_succ_eq_self_of_none`, `fullySplit_nodup_pos`
(Nodup + positivity via `List.nodup_append`; cross-subtree distinctness from
the no-squared invariant), `fullySplit_pos`, `fullySplit_unsplittable` (the
saturation lemma: `f.size ≤ fuel` ⟹ every leaf is witness-irreducible). All
public API kept signature-stable and re-proved: `factorProduct_berlekampFactor`,
`berlekampFactor_factors_ne_nil`,
`kernelWitnessSplit?_none_of_berlekampFactor_factors_length_le_one`,
`berlekampFactor_factors_nodup_of_no_squared`,
`berlekampFactor_factors_pos_degree`,
`berlekampFactor_factors_eq_singleton_of_size_le_one`,
`kernelWitnessSplit?_none_of_berlekampFactor_factors`. The CI-gated downstream
consumers (`RabinSoundness.lean`, `HexBerlekampZassenhausMathlib`,
`HexBerlekampMathlib`) reach the factor list only through membership/product,
so they carry through unchanged.

**Deliverable 3 — re-bench.** `lake exe hexbz_bench verify`: all 18
order-sensitive checksums pass with **no** `expectedHash` change (bit-identical
output). `bench run runFactorFastChecksum` split ladder improves at high
degree vs the issue baseline: deg-20 118 ms → 79 ms (~1.5×), deg-24 217 ms →
134 ms (~1.6×).

Net `-336` lines (320 insertions, 656 deletions). No `sorry`/`axiom`/
`native_decide`/`TODO`/`FIXME` added.
