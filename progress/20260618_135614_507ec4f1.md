### Key design decision (premise check)

Surjecting onto a *free* `trueSupports` is unsound: overlapping supports collapse
under `supportPartitionByMinColumn` (e.g. `{{0,1},{1,2}}` → classes `{0},{1},{2}`),
so "every `S : trueSupports` is reached by an array index" fails without a
disjointness/partition hypothesis — which deliverable 2 forbids adding. Parent
#7921's corrected deliverable resolves this: take `trueSupports` to be the
*emitted* supports. Defining `emittedSupports` as the index image makes the
index/surjectivity fact hold by construction — each member carries a witnessing
index at which the array-indexed family already produces the certificate;
transport along the support equality lands it at `S.1`. No caller hypothesis.

`(bhksProjectedRows L hrows).factorCount` reduces definitionally to
`L.factorCount` (structure-literal field), so the output's index type
(`Fin L.factorCount`) is defeq to the endpoint's expected
`Fin (bhksProjectedRows L hrows).factorCount` — directly usable, no cast.
