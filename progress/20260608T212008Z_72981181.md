# Three-tier rewire of `factor` (#6583)

## Accomplished

Rewired the public `factor` / `factorWithBound` combinator from the
two-tier `(factorFastWithBound).getD (factorSlowWithBound)` form into
the SPEC-PR-#6580 three-tier dispatch
`factorFast → factorSlowModular → factorSlowTrial`.

New names in `HexBerlekampZassenhaus/Basic.lean`:

- `factorSlowModularWithBound? : ZPoly → Nat → Option Factorization`
- `factorSlowModular?           : ZPoly → Option Factorization`
- `factorSlowModular?_eq_factorSlowModularWithBound?_default` (@[simp])
- `factorSlowModularWithBound?_eq_some_iff_safe_branch`
- `factorSlowModularWithBound?_eq_some_eq_factorSlowWithBound`

The `?` wrapper returns `some (factorSlowWithBound f B)` exactly on the
constant-core early-out, the quadratic-integer-root short-circuit, and
whenever `choosePrimeData? (normalizeForFactor f).squareFreeCore` is
`some`; otherwise `none`. The combinator routes the `none` case to
`factorSlowTrialWithBound`, so the silent prime-data fallback is
never consulted.

Case-split lemma `factorWithBound_eq_factorizationOfFactors` widened
from a two-way to a three-way disjunction (fast / slow-modular /
slow-trial), with the same shape propagated to:

- `factorWithBound_entry_mem_raw_source`,
  `factorWithBound_entry_primitive_of_chosen_raw_primitive`,
  `factorWithBound_entries_primitive`, and their default-precision
  `factor_*` specialisations;
- the Mathlib-bridge umbrellas
  `factor_entries_primitive_of_chosen_raw_primitive`,
  `factor_headline_contract_core_with_primitive`,
  `factor_entries_not_associated`,
  `factorWithBound_entry_zpolyIrreducible_of_chosen_raw_zpolyIrreducible`
  (+ `factor_entry_zpolyIrreducible_of_chosen_raw_zpolyIrreducible`),
  `factorWithBound_entries_irreducible` (+ `factor_entries_irreducible`).

`factorWithBoundUsesExhaustiveBranch` narrowed to also require
`(choosePrimeData? sf).isSome`, since under the new wiring the
no-admissible-prime case routes to the trial arm rather than to the
modular exhaustive arm. The two consumers in the same file
(`factorWithBound_entry_mem_exhaustive_branch_raw`,
`factorWithBound_entry_mem_slow_quadratic_branch_raw`) updated to
discharge the new disjunct (the trial arm contradicts the
`choosePrimeData?-some` hypothesis or the quadratic short-circuit).

`factorWithBound_product` updated to discharge the new three-tier:
`factorFastWithBound_product_of_some` / `factorSlowWithBound_product`
(via `factorSlowModularWithBound?_eq_some_eq_factorSlowWithBound`) /
`factorSlowTrialWithBound_product`. Required moving the theorem after
`factorSlowTrialWithBound_product` in the source order.
