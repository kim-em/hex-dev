## Accomplished

Closed #5253 (HexPolyMathlib Phase 6: `ofPolynomial_add` /
`ofPolynomial_mul`). Two new `@[simp]` lemmas in
`HexPolyMathlib/Basic.lean`:

- `ofPolynomial_add [Semiring R] [DecidableEq R]` — mirrors the
  existing `ofPolynomial_sub` proof shape: `ext_coeff`, then
  `Hex.DensePoly.coeff_add` with the `(0 : R) + 0 = 0` witness,
  then `simp [coeff_ofPolynomial, Polynomial.coeff_add]`.
- `ofPolynomial_mul [CommRing R] [DecidableEq R]` — one-liner
  `map_mul (equiv (R := R)).symm p q` via the existing ring
  equivalence.

Placement note: `ofPolynomial_add` lives between `ofPolynomial_sub`
and `toPolynomial_zero` as the issue specified. `ofPolynomial_mul`
necessarily lives immediately after `equiv_symm_apply`, because the
proof references `equiv` which is defined later in the file; the
issue's "insert between lines 254 and 257" instruction conflicts
with the prescribed `map_mul equiv.symm` proof, so I split the two.

Typeclass minimisation: `Semiring` for `_add` (matches
`toPolynomial_add`); `CommRing` for `_mul`, since the `equiv` ring
isomorphism is gated on `CommRing` and the issue accepted that
strengthening explicitly.
