# Work session: inline mathlib4 PR #37716 (Desnanot-Jacobi) into HexMatrixMathlib

Closes #2313 (the head of the Bareiss/HexMatrix recovery chain filed under
the human-oversight directive #2308).

## Accomplished

Created `HexMatrixMathlib/Determinant/DesnanotJacobi.lean`, inlining the
proof of the Desnanot-Jacobi identity from
https://github.com/leanprover-community/mathlib4/pull/37716 (PR HEAD
`ef2fae65896b`, originally committed by Slava Naprienko at
`bbe9ab491bc1`). The proof is copied verbatim modulo two cosmetic Lean
module-system tweaks: the upstream file's `module` declaration and
`public import` statements were replaced with plain `import` (this
project does not yet use the new module system), and the surrounding
`@[expose] public section` was dropped. All private helpers
(`succ_cast_succ_ne_zero`, …, `det_aux_p_eq`, `det_aux_adjugate_m_eq`,
the `UnivRing` / `univMatrix` / `evalMap` extension to general
commutative rings) are inlined as-is.

`HexMatrixMathlib/Determinant.lean` now `import`s the new file so that
downstream consumers (`#2314`, `#2315`) can reach `desnanot_jacobi`
through `import HexMatrixMathlib.Determinant`.

The inlined `theorem desnanot_jacobi` lives in the root namespace at
exactly the same path as upstream, which means swapping in the upstream
module after PR #37716 merges will be a one-line change (delete the
inlined file, re-point the import to
`Mathlib.LinearAlgebra.Matrix.Determinant.DesnanotJacobi`).
